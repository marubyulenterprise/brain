# 🚀 MVP 개발팀 최종 통합 실행 계획 (Final Execution Plan)

## 🎯 핵심 목표: 지식 자산의 구조화 및 소유권 확보
*   **핵심 기능:** 오디오 논리 흐름을 엔지니어링 레벨로 '구조적 블록'으로 분해하고, 이를 **Semantic Knowledge Graph** 형태로 사용자에게 제공합니다.
*   **가치 포지셔닝:** 데이터를 단순히 요약하는 것이 아니라, 사용자가 직접 편집하고 다른 시스템에 독립적으로 연결할 수 있는 *원천 지식 레이어*를 만듭니다.

## ⚙️ Phase 1: 아키텍처 및 데이터 파운데이션 구축 (Foundation)
*(의존성: 모든 후속 단계의 기반)*

| # | 마일스톤/Task | 주요 목표 산출물 | 의존성 | 담당 에이전트 | 완료 기준 (Acceptance Criteria) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1.1** | **통합 데이터 처리 레이어 설계** | 모든 원천 자산(STT, 구조화 블록)을 JSON/YAML 기반의 중립 포맷으로 저장하는 미들웨어 아키텍처 확정 (Researcher 리스크 대응). | 없음 | Developer / Secretary | *데이터 로직 흐름도*와 함께, 각 데이터를 메타태그로 품는 **'통합 스키마 정의서(Schema Definition)'**가 완료되어야 함. |
| **1.2** | **크레딧 소모 및 검증 엔진 API 구축** | 모든 데이터 입출력 트랜잭션에 대한 원자적(Atomic) 크레딧 차감/검증 엔드포인트 구현 (Developer의 기존 명세서 기반). | 1.1 | Developer | `POST /api/consume-credit`가 정상 작동하며, 성공/실패 로직이 완벽하게 분리되어야 함. |
| **1.3** | **Knowledge Graph DB 연결 및 초기화** | 데이터베이스(예: Neo4j 등)를 백엔드에 통합하고, '노드'와 '관계'를 저장할 수 있는 최소한의 CRUD API 구현. | 1.1 | Developer | 샘플 데이터를 넣어 *최소한의 지식 그래프*가 생성 및 조회되는 데모 기능 확보. |

## ✨ Phase 2: 핵심 가치 증명 엔진 구축 (Core Value Engine)

| :--- | :--- | :--- | :--- | :--- | :--- |
| **2.1** | **구조적 블록 추출 기능 고도화** | 단순 요약/변환을 넘어, 음성 데이터에서 핵심 논리 흐름(인과관계, 액션 플로우)을 구조화하여 아웃라인으로 뽑아내는 API 완성. (MVP의 메인 엔진) | 1.2 | Developer | 다양한 주제의 오디오 클립을 넣었을 때, **'논리적 흐름 다이어그램'** 형태로 변환 결과를 받을 수 있어야 함. |

| **2.3** | **독립적 자산 내보내기 기능 구현 (Export)** | 생성된 지식 그래프와 원천 데이터를 Notion, PDF 같은 외부 플랫폼에 의존하지 않고, 독립적인 JSON/Schema 형태로 다운로드할 수 있는 기능 완성. (Researcher 리스크 대응) | 1.1 | Developer | '이 파일은 우리 서비스가 만든 것'이라는 느낌을 줄 만큼, 포맷과 메타데이터가 완벽하게 유지된 자산물이 나와야 함. |
