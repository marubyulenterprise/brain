## 📄 PVU Calculator MVP - 엔드-투-엔드 사용자 워크플로우 스펙
**목표:** 사용자가 자신의 프로세스 내 'Missing Edge'를 발견하고, 놓치고 있는 잠재 수익(PVU)의 크기를 정량적으로 인지하여 다음 단계 진입을 요청하도록 유도한다.
**톤앤매너:** 엔지니어다운 정밀함 + 경제적 절실함 (데이터 기반 위기감 조성).

### 🟢 Phase 1: 데이터 입력 및 초기 상태 (The Input Stage)

| State/화면 | 컴포넌트 | 액션/로직 트리거 | 카피라이팅 (Copy) | 미세 문구 (Microcopy) / 가이드 |
| :--- | :--- | :--- | :--- | :--- |
| **1. Hero Section** (랜딩 페이지 진입점) | Headline | — | 🛑 당신의 비즈니스 프로세스에 숨겨진 돈이 있습니다. PVU를 계산하세요. | *Sub-Headline:* "단순한 데이터 분석을 넘어, A(자원) $\times$ B(관계성) $\times$ C(시장의 Edge) 공식으로 잠재 수익 기회를 발견합니다." |
| **2. Input Form** (A/B/C 입력 페이지) | 설명 텍스트 | 사용자가 어떤 데이터를 넣어야 하는지 안내. | "프로세스를 분석하여 놓치고 있는 '잠재 가치'를 찾아보세요." | *가이드:* 각 항목은 명확한 비즈니스 프로세스 단계와 연결되어야 합니다. (예: 고객 온보딩 $\to$ 데이터 수집) |
| **2-1. A 입력** | 텍스트 필드 / 설명 | '자원'의 종류(시간, 인력, 데이터셋 등)를 정의하게 함. | **[A] 활용 가능한 핵심 자원 (Resource):** 어떤 자원을 갖고 있나요? | *예시:* "지난 분기 고객 문의 로그", "팀원의 전문 기술 스택 목록" 등 구체적인 자산으로 입력 유도. |
| **2-2. B 입력** | 텍스트 필드 / 설명 | '관계성' 또는 '연결 고리'를 정의하게 함. (가장 중요) | **[B] 발견 가능한 관계성/연결점 (Edge):** 이 자원들 사이에 어떤 숨겨진 연결이 있나요? | *힌트:* "단순한 데이터 나열이 아닌, A와 A 사이의 상호작용(Interaction)을 정의하세요." (예: '문의 로그' $\to$ '기술 스택'으로의 연관성) |
| **2-3. C 입력** | 텍스트 필드 / 설명 | 시장의 요구사항/미발견 가치를 정의하게 함. | **[C] 놓치고 있는 시장 기회 (Opportunity):** 이 조합이 현재 시장에서 어떤 '결핍'을 채울 수 있나요? | *강조:* "가장 중요한 것은, 아직 아무도 주목하지 않은 비효율성이나 결핍(Pain Point)입니다." |
| **2-4. 실행 버튼** | CTA Button | 모든 필드가 유효한 값을 갖췄는지 검증 후 계산 로직 트리거. | **[CTA] Missing Edge 발견하고 PVU 계산하기 (무료 진단)** | *Microcopy:* "데이터 입력 완료. 다음 단계에서 귀하의 잠재 가치를 정량적으로 측정합니다." |

### 🟠 Phase 2: 결과 및 골드 경고 발생 (The Revelation Stage)

**(로직 트리거: API 호출 성공 / PVU 계산 로직 실행)**
**결과 상태:** **🔴 Gold Alert (매우 높음)**

| State/화면 | 컴포넌트 | 액션/로직 트리거 | 카피라이팅 (Copy) | 미세 문구 (Microcopy) / 가이드 |
| :--- | :--- | :--- | :--- | :--- |
| **3. Result Screen** (최초 진입) | 메인 시각 요소 | PVU 수치와 그래프를 가장 크고 눈에 띄게 배치. (골드 배경/강조색 사용 필수) | **🚨 경고: 당신의 잠재 수익(PVU)은 [XXX]원 규모입니다.** | *헤드라인:* "현재 비즈니스 프로세스에서는 이 가치를 놓치고 있습니다." (손실 기회 강조) |
| **3-1. PVU 설명** | 텍스트 블록 | 계산된 수치가 의미하는 바를 기술적으로 설명. | **[PVU = A $\times$ B $\times$ C]의 분석 결과:** 귀하가 보유한 [A 자원]과 [B 관계성]을 시장의 미발견 기회(C)에 연결할 경우, 최소 연간 \[XXX]원의 추가 순이익 확보 가능성이 확인되었습니다. | *강조:* 이 숫자는 '가능성'이 아닌, **논리적 구조 분석 기반의 잠재 수익**입니다. (기술적 신뢰감 부여) |
| **3-2. Missing Edge 시각화** | 다이어그램/차트 | A와 B 사이에 발견된 연결고리를 명확히 보여줌. | **발견된 'Edge'**: 기존에는 [A]를 개별적으로 사용했지만, 실제로는 \[B]라는 연결점을 통해 폭발적인 가치를 창출할 수 있습니다. | *전환:* "이 Edge가 곧 새로운 수익 파이프라인입니다." (Problem $\to$ Solution) |
| **3-3. 위기감 조성** | 텍스트 블록 | 사용자가 현재 아무것도 하지 않았을 때의 손실을 명확히 제시. | 이 잠재 가치를 인지하고도 액션하지 않으면, 기회는 곧 비용(Cost)으로 변환됩니다. | *CTA 근접 Microcopy:* "이 분석은 시작에 불과합니다. 실행 가능한 구체적 로드맵이 필요합니다." (다음 스텝 유도) |

| **4-1. 솔루션 소개** | 설명 섹션 | Maru Company의 역할(계획, 도구 제공) 정의. | 저희는 단순한 진단 툴이 아닙니다. 귀하의 프로세스에 최적화된 **[실행 가능한 로드맵]을 설계하는 AI 에이전트 팀**입니다. | *원리 요약:* "막연한 이론 대신, 당장 오늘 적용할 수 있는 '체크리스트'와 'API 연결 시나리오'를 제공합니다." (기술적 신뢰성) |
| **4-2. 진단 폼 유도** | 핵심 CTA Block | 사용자에게 구체적인 정보 입력(진단 폼 작성)을 요구하며 가치를 명확히 제시. | **🚀 [클릭] 나만의 맞춤형 PVU 실행 로드맵 받아보기 (무료)** | *가치 강조:* "이 진단을 통해 \[X월 Y일]까지 최소 \[최소 예상 금액]의 추가 수익 창출 가능성을 확인하고, 구체적인 액션 플랜을 받게 됩니다." |
| **4-3. 결론 Microcopy** | 마지막 안내 문구 | 사용자가 망설이는 순간을 포착하여 행동을 촉진. | 막연한 '좋은 아이디어'는 비용이 들지만, 저희가 제시하는 이 로드맵은 검증된 시스템 기반의 설계도입니다. 지금 바로 다음 단계로 넘어가세요. | *CTA 대비:* (버튼 위에 작은 글씨) "신용카드나 결제 정보는 필요하지 않습니다. 오직 분석에 필요한 정보만 입력해주세요." (심리적 장벽 제거) |
