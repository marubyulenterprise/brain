## ✨ [2주차] MVP 핵심 기능 개발 스프린트 플랜 (Jira Style)

**🎯 목표:** PVU 기반 'Missing Edge 예측 경고' 기능을 구현하고, 사용자에게 실질적인 비즈니스 액션(CTA)을 유도하는 통합 인터페이스 완성.
**🛠️ 주력 에이전트:** 코다리 (Developer)
**🗓 기간:** 2026년 5월 19일 ~ 5월 30일 (가정)

### 🟢 Week 1: 데이터 구조 및 기본 경고 구현 (Foundation & Basic Alert)
*이번 주 목표는 API의 핵심 로직을 완성하고, 가장 일반적인 'Discovery Alert' 컴포넌트를 개발하는 것입니다.*

| 카테고리 | 태스크 ID | 제목/태스크 설명 | 담당 | 예상 소요 시간 | Acceptance Criteria (AC) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Backend** | `BE-101` | **[API] Missing Edge 예측 엔드포인트 구현**: `/api/predict-missing-edge` (POST). 핵심 입력값(Context ID, Current Concepts)을 받아 PVU를 계산하고 3단계 Alert 데이터를 반환해야 함. | 코다리 | 2일 | - API 호출 시 정상 응답 코드(200 OK)가 나와야 함.<br>- `PVU_Value` (Float)와 `Alert_Level` (Enum: DISCOVERY, OPPORTUNITY, CRITICAL) 필드가 포함되어야 함. |
| **Backend** | `BE-102` | **[Logic] PVU 계산 로직 통합 테스트**: 콘텐츠 분석 스키마를 기반으로 잠재적 연결고리(Missing Edge)의 가치를 정량화하는 핵심 백엔드 로직 구현 및 단위 테스트 작성. | 코다리 | 1일 | - *Mock 데이터* 입력 시, 경고 레벨에 맞는 PVU 값이 정확히 계산되어야 함.<br>- 트랜잭션 실패 시 원자적 롤백(Atomic Rollback)이 정상적으로 작동해야 함. |
| **Frontend** | `FE-201` | **[Component] AlertCard 기본 구조 구현**: Missing Edge 경고를 담는 재사용 가능한 UI 컴포넌트(`AlertCard`)의 기본적인 레이아웃 및 스타일링 (앰버/골드 톤). | 코다리 | 1.5일 | - 카드 제목, 본문 설명(Text), PVU 그래프 Placeholder, CTA 버튼 영역이 모두 포함되어야 함.<br>- 디자인 가이드라인에 정의된 컬러 팔레트가 적용되어야 함. |

### 🟡 Week 2: 고급 시나리오 통합 및 완성 (Advanced Integration & Polish)
*이번 주 목표는 나머지 두 가지 경고 시나리오(Opportunity, Critical)를 API와 연동하고, 최종 사용자 경험을 다듬는 것입니다.*

| 카테고리 | 태스크 ID | 제목/태스크 설명 | 담당 | 예상 소요 시간 | Acceptance Criteria (AC) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Backend** | `BE-201` | **[API] 시나리오별 데이터 처리 로직 강화**: Discovery 외에, Opportunity(예측적 기회) 및 Critical(위험 임계치 초과) 시나리오에 따른 세부 데이터를 API 응답 구조에 추가하고 분기 처리를 완성해야 함. | 코다리 | 1.5일 | - 요청된 `Alert_Level`에 따라 적절한 데이터 필드와 경고 메시지가 반환되어야 함.<br>- **AC:** Critical 레벨 시, 시스템 에러가 아닌 '즉시 액션'을 유도하는 데이터를 성공적으로 전송해야 함. |
| **Frontend** | `FE-202` | **[Integration] 3단계 경고 UI/UX 구현**: `AlertCard` 컴포넌트에 Opportunity 및 Critical 레벨의 특화된 시각 효과(Pulse & Flicker 애니메이션, 컬러 코딩)를 적용하고 동적 상태 변화를 테스트함. | 코다리 | 2일 | - **AC:** API에서 전달된 `Alert_Level`에 따라 카드의 배경색, 아이콘 색상, 깜빡임 패턴이 정확히 변해야 함.<br>- CTA 버튼의 활성화/비활성화 로직(데이터 기반)을 구현해야 함. |
| **Full Cycle**| `TC-301` | **[Testing] 통합 E2E 테스트 및 리팩토링**: 백엔드 API와 프론트엔드 컴포넌트를 연동하여, 3가지 시나리오 전체 흐름에 대한 사용자 스토리 기반 End-to-End 테스트를 수행하고 코드 정리. | 코다리 | 0.5일 | - 모든 경고가 성공적으로 표시되며, 버튼 클릭 시 다음 액션(예: 관련 콘텐츠 탐색)으로 연결되는 경로가 끊김 없이 작동해야 함.<br>- 전체 코드의 가독성 및 재사용성이 확보되어야 함. |
