# 📄 Pro Tier MVP 핵심 기능 요구사항 명세서 (V1.0)
**문서 목적:** 지식 그래프 빌더의 수익화 최소 단위(MVP Flow)를 정의하고, 각 흐름별 기술적 스코프와 크레딧 소모 규칙을 확정합니다.

## 🚀 Pro Tier 가치 핵심: '분산된 자산의 통합 및 연결'
*   **Basic/Free:** 단순 변환 (1차원 구조화).
*   **Pro Tier:** 다중 원천 데이터를 입력받아 상호 참조하는 지식 그래프 구축 (3차원 연결).

---

## 💡 Flow 1: [기본 자산화] 오디오/텍스트 기반 핵심 개념 추출 및 노드 생성
가장 기본이 되는 단위 작업입니다. 사용자가 단일 원천 콘텐츠를 입력하여 가장 기본적인 '지식의 최소 단위'를 뽑아내는 과정입니다. (예: 회고록 1편 -> 주요 개념 리스트)

| 항목 | 상세 요구사항 | 기술적 정의 및 스코프 |
| :--- | :--- | :--- |
| **입력 데이터 종류** | 원천 콘텐츠 파일 (Audio/Video Transcripts, Long-form Text Docs). | `SourceFile` (텍스트 또는 시간 정보가 포함된 텍스트) |
| **핵심 목표 가치** | 비정형 데이터를 구조화 가능한 개별 개념(Concept Node)으로 변환. | '요약'이 아닌, *개념 단위의 추출*에 집중. |
| **시스템 처리 로직 순서** | 1. **STT/텍스트 입력 받기:** 원본 콘텐츠를 수신.<br>2. **세그멘테이션 (Segmentation):** 논리적 흐름(단락, 주제)별로 분할.<br>3. **개체명 인식 (NER):** 핵심 키워드 및 인물/사건을 추출하여 잠정적인 `Node` 생성.<br>4. **초기 구조화:** 각 Node를 기반으로 제목-요약 쌍의 초기 텍스트 자산 출력. | $SourceFile \xrightarrow{\text{STT}} [\text{Segment}] \xrightarrow{\text{NER}} \{\text{Concept Nodes}\} \rightarrow \text{Structured Asset}$ |
| **크레딧 소모량 (예상)** | **3~5 크레딧** / 1회 처리당. (입력 데이터 길이 및 세그먼트 수에 비례하여 가변적) | *가장 낮은 비용으로, 사용자가 가장 많이 시도하게 만드는 유인책.* |

## 🔗 Flow 2: [지식 연결] 다중 원천 간의 관계형 지식 그래프 구축 (핵심 차별점)
Pro Tier의 핵심이자 가장 높은 WTP를 반영하는 플로우입니다. 여러 개의 개별 자산(Flow 1의 결과물)을 입력받아, 그들 사이의 논리적 **관계(Edge)**를 발견하고 연결합니다. (예: A 프로젝트 보고서 + B 경쟁사 분석 자료 → 'A가 B에 적용할 수 있는 방법론'이라는 새로운 Edge 생성)

| 항목 | 상세 요구사항 | 기술적 정의 및 스코프 |
| :--- | :--- | :--- |
| **입력 데이터 종류** | **구조화된 자산의 집합 (Collection of Structured Assets).** (Flow 1 또는 다른 사용자/외부 자산) | `Asset_A`, `Asset_B`, `Asset_C` ... 등 N개. |
| **핵심 목표 가치** | 개별 정보 조각들을 연결하여 '지식 네트워크'를 완성하고, 누락된 연결고리(Gap Analysis)를 제안. | 단순 취합이 아닌, *시너지*와 *가설 도출*을 통해 새로운 지식을 창조함. |
| **시스템 처리 로직 순서** | 1. **자산 수신:** N개의 구조화된 자산을 입력받음.<br>2. **유사성/대조 분석 (Similarity & Contrast):** 각 자산 쌍 간의 공통 주제, 상반 개념을 비교 검색.<br>3. **관계 정의 및 매핑 (Relationship Mapping):** 발견된 관계를 `Edge`로 정의하고, 이 Edge에 대한 설명을 자동 생성.<br>4. **지식 그래프 출력:** 노드-엣지 구조를 시각화 가능한 형태로 최종 출력. | $\{\text{Asset}_1 \dots \text{Asset}_N\} \xrightarrow{\text{Compare}} \{\text{Edges}\} \rightarrow \text{Knowledge Graph}$ |
| **크레딧 소모량 (예상)** | **20~40 크레딧** / 1회 처리당. (입력 자산의 개수 $N$에 비례하여 급격히 증가) | *가장 높은 비용을 책정하여, 이 플로우를 사용해야만 Pro Tier 가치가 느껴지게 만듭니다.* |

## ✨ Flow 3: [최종 활용] 구조화된 지식을 목적 기반 아웃풋으로 전개
지식 그래프 자체에 머무르지 않고, 이를 실제 '결과물'로 즉시 사용할 수 있게 만드는 플로우입니다. (예: 완성된 KB를 바탕으로 투자자 발표용 슬라이드 초안 생성)

| 항목 | 상세 요구사항 | 기술적 정의 및 스코프 |
| :--- | :--- | :--- |
| **입력 데이터 종류** | 최종 지식 그래프 (`Knowledge Graph` structure). | `Graph_Output`. (Flow 2의 결과물) + 사용자 지정 목표/템플릿. |
| **핵심 목표 가치** | 복잡한 구조를 사용자가 원하는 형식(Presentation, Report Outline, Blog Draft 등)에 맞게 자동 변환하고 정제. | 지식 자산 $\rightarrow$ *실행 가능한 문서*로의 전환 (Actionability). |
| **시스템 처리 로직 순서** | 1. **목표 정의:** 사용자에게 아웃풋 목적(예: 투자 유치, 내부 보고)을 명시받음.<br>2. **템플릿 매칭:** 선택된 목표에 맞는 구조적 프레임워크(Template)를 불러옴.<br>3. **데이터 추출 및 배치 (Extraction & Placement):** 그래프 내의 노드와 엣지 중 해당 템플릿 요구사항에 맞는 데이터를 선별적으로 뽑아냄.<br>4. **문체 교정/확장:** 단순히 붙여넣는 것이 아닌, 목적에 맞는 문맥과 논리적 연결어(Transition Phrase)를 추가하여 완성도를 높임. | $\text{Graph} + \text{Template} \xrightarrow{\text{Select}} \{\text{Text Output}\} \rightarrow \text{Final Document}$ |
| **크레딧 소모량 (예상)** | **5~10 크레딧** / 1회 처리당. (최종 결과물의 분량에 비례) | *사용자가 최종적으로 '가치를 얻었다'고 느끼게 만드는 마무리 단계.* |
