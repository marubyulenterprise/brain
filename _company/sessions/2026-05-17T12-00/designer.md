# 💡 Missing Edge Shock Component: 인터랙티브 프로토타입 와이어프레임 스펙 V6.0 (Technical Spec)

## I. 개요 및 목표
*   **목표:** 사용자가 데이터의 '단순 결핍'을 넘어, **'재무적 손실액(Loss Amount)'**이라는 정량화된 충격으로 자각하고, 이를 해결할 잠재적 기회(PVU)를 명확히 인지하게 한다.
*   **핵심 원칙:** 기술 스펙($V6.0$)은 $SAGA$ 트랜잭션의 성공/실패 여부에 따라 UI 컴포넌트의 상태가 완전히 변하는 **상태 기계(State Machine)**로 작동한다.
*   **API 의존성:** 본 디자인은 `shock-detector` Mock API Gateway (Kodari)가 정의한 Input/Output Schema와 트랜잭션 시퀀스에 전적으로 의존한다.

## II. 데이터 흐름 및 상태 기계 매핑
| State | 단계 명칭 | 사용자 심리 상태 | API 동작 및 결과 | 주요 UI 변화 |
| :---: | :--- | :--- | :--- | :--- |
| **S0** | 초기/정상 관찰 (Baseline) | 무관심 $\to$ 호기심 | 🟢 `GET /api/shock-detector` 호출 전. 로컬 데이터만 표시. | 일반적인 지식 그래프, 평온한 색감(Blue Tone). |
| **S1** | 위기 충격 자각 (Shock Detection) | 불안 $\to$ 경고 (Panic) | 🟡 `GET /api/shock-detector` 호출 시작. Graph $\to$ Finance 데이터 수신 중 실패 감지. | 배경의 노이즈 증가, 카운터 폭주, 빨간색(Danger Red). **핵심 충격 발생.** |
| **S2** | 손실액 확정 (Loss Quantification) | 공포 $\to$ 인식 (Realization) | 🔴 `GET /api/shock-detector` 트랜잭션 실패. Loss Amount($L$) 계산 완료 및 출력. | UI 중앙에 $L$ 표시, 데이터 파편화 시각화, 강렬한 경고음(사운드 로직). |
| **S3** | 통합 기회 발견 (PVU Opportunity) | 해결 욕구 $\to$ 자신감 | 🟢 `POST /api/shock-detector/predict` 호출. Missing Edge $E_{miss}$ 탐지 성공 및 PVU 제시. | 색상이 금색(Gold)으로 전환, 구조화된 CTA 배치, 명확한 액션 플랜 제시. **제품 USP 경험.** |

## III. 컴포넌트별 상세 인터랙션 스펙 (Timeline & Logic)
*(타이밍은 초 단위로 계산되었으며, 모든 애니메이션은 부드러운 커브(Ease-out/in-out)를 사용합니다.)*

### 🔴 State S1: 위기 충격 자각 (Shock Detection Phase) - [시간: 0.0s ~ 2.5s]
**트리거:** 백그라운드에서 `Graph`와 `Finance` 데이터를 가져오던 Mock API 호출이 실패하거나, 핵심 데이터의 **결핍(Missing Edge)**이 감지되는 순간.

1.  **[T=0.0s] - 트리거 발생:** 화면 전체에 미세한 노이즈 패턴(Grainy noise)이 퍼지기 시작하며 배경색이 짙은 회색조로 전환된다.
2.  **[T=0.5s] - 데이터 충돌 시각화:** 그래프 상의 연결된 엣지들이 갑자기 **Pulse & Flicker 애니메이션**을 반복하며 깜빡인다 (빨간색 $\to$ 주황색). 이는 "데이터 흐름에 문제가 있다"는 무언의 경고를 준다.
3.  **[T=1.0s] - 충격 임계점 도달:** 화면 중앙 하단에 **'ALERT: Critical Data Gap Detected'**라는 텍스트가 강렬하게 플래시(Flash)된다. 이와 동시에 `Loss Amount` 카운터 모듈이 활성화된다.
4.  **[T=1.0s $\to$ 2.5s] - 폭발적 손실액 증가 (The Core Shock):**
    *   카운트 애니메이션은 단순 증감이 아니라, **Exponential Curve (지수 곡선)**를 따르며 가속한다. ($L_{t} = L_{initial} \cdot e^{k(t-1)}$).
    *   디자인 원칙: 숫자가 팝업되는 것이 아니라, 마치 **압력계가 터지는 것처럼** 빠르게 수치를 쌓아 올리며 카운터의 최댓값에 도달한다.
    *   이 기간 동안 UI 전체는 강한 경고음과 함께 빨간색/주황색 대비를 유지하며 긴장감을 최고조로 끌어올린다.

### 🟡 State S2: 손실액 확정 및 파편화 (Loss Quantification Phase) - [시간: 2.5s ~ 4.0s]
**트리거:** `Finance`와 `Edge` 데이터 분석이 완료되어 최종 $L$ 값이 산출되는 순간.

1.  **[T=2.5s] - 충격 정지 및 고정:** 카운팅 애니메이션이 갑작스럽게 멈추고, 화면 중앙에 **최종 손실액($Loss Amount$)**이 매우 크고 무게감 있는 타이포그래피로 고정된다 (가장 중요한 요소).
2.  **[T=3.0s] - 데이터 파편화 시각화:** 기존의 깔끔했던 지식 그래프(Nodes & Edges)는 깨지고, $L$에 비례하는 양만큼의 **'데이터 조각(Data Shards)'**들이 화면을 맴돌며 불안하게 떠다닌다. 이는 사용자의 '정보가 휘발되고 있다'는 심리적 상태를 시각화한다.
3.  **[T=4.0s] - 전환 준비:** 전체적인 색감이 빨간색의 긴장감에서, 다음 단계(해결책)를 암시하는 **어두운 골드 톤**으로 점진적으로 변조되기 시작한다.

### 🟢 State S3: 통합 기회 발견 (PVU Opportunity Phase) - [시간: 4.0s ~ End]
**트리거:** `Predictive Alert` 로직이 성공하여 'Missing Edge'가 포착되고, $E_{miss}$에 대한 PVU가 계산되어 사용자에게 제시되는 순간.

1.  **[T=4.0s - 5.0s] - 드라마틱한 전환:** 데이터 파편화(S2)를 만들던 노이즈와 빨간색 배경이 갑자기 **안정적이고 구조적인 골드/네이비 톤**으로 클린하게 정리된다. (Visual Reset).
2.  **[T=5.0s] - Missing Edge 하이라이트:** 화면 중앙의 지식 그래프가 재구성되면서, $L$을 발생시켰던 'Missing Edge' 자리에 **새롭고 빛나는 노드(Potential Node)**가 생성된다. 이 과정은 마치 스위치를 켜는 것처럼 매끄러워야 한다.
3.  **[T=5.0s - End] - 행동 유도 (CTA):**
    *   Missing Edge에 대한 설명과 함께, **"이 기회를 포착하고 추가 수익을 얻으세요."**라는 메시지가 제시된다.
    *   가장 중요한 CTA 버튼: `✅ Missing Edge를 인덱싱하여 PVU 확보하기` (골드 배경의 굵고 명확한 버튼). 이 버튼은 사용자가 가장 쉽게 클릭할 수 있도록 시각적 계층 구조 최상단에 배치되어야 한다.
