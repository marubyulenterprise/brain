# 💡 [Component Spec] Missing Edge & PVU Alert System (Master Component)

## 1. 개요 및 목표
이 컴포넌트는 사용자의 데이터 흐름(Data Flow)에서 발견된 연결성 부족(Missing Edge)을 단순히 '정보 누락'으로 처리하는 것이 아니라, **정량화 가능한 '재무적 기회비용 손실액($)'**으로 변환하여 사용자에게 제시하는 것을 목표로 한다.
*   **USP:** 단순 경고 $\rightarrow$ 재무적 위협(Loss) $\rightarrow$ 즉각적인 행동 유도(Value).
*   **핵심 원칙:** '놓치고 있는 돈의 크기'라는 감정적 충격과, 이 문제를 해결할 수 있다는 기술적 확신을 동시에 부여해야 한다.

## 2. 상태별 시각 디자인 규칙 (State & Color Token)

| 상태 | 이름 | 목적/메시지 | 색상 토큰 | 애니메이션 원칙 |
| :--- | :--- | :--- | :--- | :--- |
| **① 경고** | Missing Edge Alert | 데이터 연결 부족. 누락된 정보의 종류와 규모 제시. (정보성) | `Amber: #FFC72C` | 부드러운 진동(Pulse Effect). 주변 영역이 흐릿하게 떨리는 효과 적용. |
| **② 기회 예측** | PVU Opportunity Alert | 연결할 경우 예상되는 추가 순이익(PVU) 제시. (가치성) | `Gold: #FFC72C` $\rightarrow$ *강조* | 점진적 발광(Flicker Glow). 숫자가 카운트업 되며 나타나야 함. **(최우선 강조)** |
| **③ 위험 임계치** | Critical Action Alert | 이대로 방치할 경우의 재무적 손실이 심각함. 즉시 액션 필요. (위기감) | `Red: #DC2626` | 강한 경고음(Haptic/Audio Cue)과 함께 깜빡임(Blinking). 배경에 붉은 그림자 오버레이. |

## 3. 인터랙션 상세 와이어프레임 및 애니메이션 스토리보드 (The Loss Calculation Flow)
이 과정은 사용자가 'Missing Edge'를 발견하는 순간부터 PVU가 최종 계산되어 제시될 때까지의 **핵심 UX 플로우**이다. 총 3단계의 시각적 전환을 거쳐야 한다.

### Step 1: Pain Point 인식 (Input & Context Setting)
*   **Trigger:** 사용자가 데이터를 입력하거나, 시스템이 분석 결과를 반환할 때.
*   **Visuals:** 화면의 좌측 상단에 '데이터 흐름 그래프(Knowledge Graph)'가 그려지며 특정 노드 A와 B 사이에 공백(Missing Edge)이 감지된다는 애니메이션을 보여준다.
*   **Animation Rule:** Missing Edge 지점은 **진한 회색 점선**으로 표시되며, 해당 선 주변의 데이터 연결 라인들이 *끊어지는 듯한 (Broken)* 시각적 효과를 준다.
*   **Micro-Interaction:** 'Missing Link' 위에 마우스를 올리면(`hover`), 팝오버가 나타나 "데이터 A와 B 간의 관계성(Edge)이 확인되지 않았습니다." 라는 문구가 표시된다.

### Step 2: 기회비용 손실액 계산 과정 (The Loss Calculation - Critical Moment)
*   **Trigger:** 사용자가 'Missing Edge'를 클릭하거나, 시스템이 자동 분석을 시작할 때.
*   **Visuals:** 화면 중앙에 **'PVU 예측 엔진 작동 중...'** 이라는 로딩 인터페이스가 나타난다. 이때, 기존의 단순 경고(Amber)에서 황금빛(Gold)으로 색상 톤이 급격히 바뀐다.
*   **Animation Rule (The Calculation):**
    1.  세 개의 변수 박스($A \times B \times C$)가 순차적으로 활성화된다. 각 변수는 데이터의 종류(예: 고객 피드백, 구매 이력, 산업 트렌드)를 상징하며, 값은 0에서 시작해 **카운트업 애니메이션**을 거쳐 실제 값이 채워진다.
    2.  변수들이 결합되는 과정(A $\rightarrow$ B)에서는 빛의 파동이 일어나며 '연결 자산 가치'가 증폭되는 시각 효과를 준다. (기술적 설명 대신, **마법적인/신뢰감 있는** 애니메이션 사용).
    3.  최종적으로 계산된 값이 가장 큰 폰트 크기로 중앙에 나타나기 직전에 잠시 *정지(Hold)* 상태가 된다.

### Step 3: PVU 결과 제시 및 액션 유도 (Output & CTA)
*   **Trigger:** 계산이 완료되어 최종 값이 산출되었을 때.
*   **Visuals:** 가장 중요한 영역이다. 화면 전체에 **황금빛 오버레이(Golden Flare)**가 잠시 감돌며 시선을 집중시킨다.
*   **핵심 출력 (The Punchline):** 중앙에 계산된 최종 금액이 카운트업 애니메이션과 함께 나타난다.
    > 💰 **놓치고 있는 기회비용 손실액:** <span style="font-size:3em; color:#DC2626;">₩4,500,000</span> (최소 예상)
*   **CTA Placement & Design:** 최종 금액 바로 아래에 CTA 버튼이 가장 큰 시각적 무게감으로 배치된다.
    *   **버튼 텍스트:** "PVU 연결 자산 확보하기 / 지금 액션 플랜 보기"
    *   **위치:** 반드시 '놓치고 있는 돈의 크기'라는 수치 바로 아래에 위치하여, 사용자가 감정적 충격을 받은 상태에서 즉각적인 해결책을 찾도록 유도한다.
    *   **마이크로 인터랙션:** CTA 버튼은 마우스를 올릴 때(hover) 빛나며(`Gold Glow`), 클릭 시 활성화되는 애니메이션이 명확해야 한다.

## 4. 개발자 가이드라인 요약 (Developer Blueprint)
1.  **반응성:** 모든 단계는 모바일(375px 기준)에서도 동일한 중요도와 크기로 인식되어야 한다. '손실액' 수치는 어떤 디바이스에서든 가장 눈에 띄어야 한다.
2.  **트랜잭션:** PVU 계산 과정은 비동기(`Async`) API 호출로 처리하고, 로딩 상태는 Step 2의 애니메이션을 통해 사용자가 기다림을 인지하도록 해야 한다. (Spinner X - 스토리텔링 Y)
3.  **가독성:** 재무적 수치($/₩)가 포함된 모든 텍스트에는 `Prose-Bold` 스타일과 `Gold: #FFC72C`의 보조 색상을 사용하여 중요도를 높여야 한다.
