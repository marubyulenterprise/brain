# 💰 Maru Company 수익화 전략 제안 (3가지 Model)

## 🎯 핵심 전제: 가치 판매 단위의 정의
우리가 파는 것은 *기능*이 아니라, **`[Pain Point] → [데이터 구조화] → [잠재적 수입/효율성 발견]`**으로 이어지는 완전한 '가치 흐름'입니다. 따라서 모든 가격 모델은 이 과정 중 가장 높은 효용을 얻는 지점(즉, Edge Discovery)에 비용이 책정되어야 합니다.

---

### 💡 전략 A: 하이브리드 크레딧 모델 (Hybrid Credit Model)
*   **적합 고객:** 초기 진입 장벽을 낮추고, 다양한 사용 패턴을 테스트하는 일반 1인 창업가/프리랜서.
*   **결제 로직:** Freemium(무료) 기반으로 유입시키되, 핵심 가치 발견 시 크레딧 소모를 강제하여 과금화합니다.

| 항목 | 내용 | 상세 설명 |
| :--- | :--- | :--- |
| **초기 진입 장벽** | **매우 낮음 (Freemium)** | 초기 계정 생성 및 기본 기능(예: 콘텐츠 구조화 3회)은 무료로 제공하여 사용자가 도구에 익숙해지도록 합니다. |
| **핵심 가치 전달 방식** | **사용량 기반 제한과 희소성 강조** | 'Missing Edge 발견' 같은 핵심 결과물(Outcome)을 크레딧으로 전환합니다. 무제한의 잠재 수익은 존재하지만, 그것을 *증명*하는 행위에는 비용이 듭니다. |
| **최소 단위 가격 책정 기준** | **크레딧 패키지 (Consumption Unit)** | 핵심 기능(PVU 계산, 지식 그래프 변환) 실행 건당 크레딧 차감. <br>예: '고급 Edge Discovery 분석' 1회 = 5 크레딧. |
| **강점/약점** | ✅ 낮은 이탈률, 높은 확장성 / ❌ 매출 예측의 복잡성 (사용량에 따라 변동) |

### ✨ 전략 B: 구독 + 성공 보장 모델 (Subscription + Outcome Guarantee)
*   **적합 고객:** 안정적인 소득 흐름이 필요한 전문직(컨설턴트, 작가 등), 고품질 결과물을 원하는 사용자.
*   **결제 로직:** 월 정액을 통해 지속적인 사용 권한과 '안심'이라는 감정적 가치를 판매합니다.

| 항목 | 내용 | 상세 설명 |
| :--- | :--- | :--- |
| **초기 진입 장벽** | **중간 (월 구독료 요구)** | MVP에 대한 신뢰를 구축하고 일정 수준의 재무 안정성을 확보하기 위해 초기 결제가 필요합니다. |
| **핵심 가치 전달 방식** | **'결과물의 품질 보장' 및 '시간 절약' 판매** | 단순히 기능을 많이 쓰는 것이 아니라, 월 구독료만큼 *최소한의 잠재적 수익(PVU 임계값)*을 발견할 수 있다는 것을 약속합니다. (예: "매월 최소 50만원 이상의 미발견 가치를 찾아드립니다.") |
| **최소 단위 가격 책정 기준** | **Tiered Monthly Subscription (Fixed Fee)** | 3단계 구독 모델(Basic, Pro, Enterprise)을 설정하고, 각 티어별로 제한된 크레딧/사용량을 포함합니다. <br>예: Basic ($19/월 - 50 크레딧), Pro ($49/월 - 무제한 분석 + 우선 지원). |
| **강점/약점** | ✅ 예측 가능한 안정적 월 매출 확보 / ❌ 고객의 '실질적인 효용'이 명확하지 않으면 이탈률 높음 |

### 🚀 전략 C: 용량 기반 라이선스 모델 (Capacity Licensing Model)
*   **적합 고객:** 데이터가 끊임없이 발생하는 대규모 사용자, 소형 에이전시, 또는 마케팅 팀. (B2B 기업 시장 지향)
*   **결제 로직:** 서비스 사용 횟수나 기능에 관계없이, 시스템을 돌리는 '용량' 자체를 구매하게 합니다.

| 항목 | 내용 | 상세 설명 |
| :--- | :--- | :--- |
| **초기 진입 장벽** | **높음 (견적 및 계약 필요)** | 사용자 규모와 데이터 처리량을 파악하는 컨설팅 과정이 필수적이므로, 초기 상담과 견적이 필요합니다. |
| **핵심 가치 전달 방식** | **'통합 시스템으로서의 역할' 강조** | Maru Company를 단순 도구가 아니라, 회사의 워크플로우에 완전히 녹아들어 '자산화된 지식 베이스 운영체제(OS)'로 포지셔닝합니다. |
| **최소 단위 가격 책정 기준** | **사용자 좌석 수 + 데이터 처리량 (Seat + Volume)** | 월별 사용자 라이선스 비용에 더해, 처리되는 원천 데이터의 총 볼륨(예: 10TB/월)을 기준으로 추가 요금을 부과합니다. <br>예: Base Seat Fee + Data Processing GB당 $X. |
| **강점/약점** | ✅ 높은 ARPU(Average Revenue Per User), 안정적 대형 계약 / ❌ 진입 장벽이 높고, 초기 고객 확보 속도가 느림 |

## 📊 현빈의 최종 권고 (Recommendation)

현재 MVP 단계이고 시장 검증을 최우선으로 한다면, **전략 A: 하이브리드 크레딧 모델**로 시작해야 합니다.

1.  **검증 용이성:** 무료 사용자를 통해 실제로 어떤 기능(A, B, C 중 어느 것이 가장 자주 쓰이는지)에 대한 지불 의향(WTP)이 높은지 데이터를 수집할 수 있습니다.
2.  **시장 침투력:** 낮은 진입 장벽 덕분에 초기 사용자 풀을 빠르게 확보하여 제품-시장 적합성(PMF) 검증 속도가 빠릅니다.

**다음 단계 액션 플랜:** 전략 A를 중심으로 MVP의 결제 시스템을 설계하고, 크레딧 소모 로직을 가장 먼저 확정해야 합니다.
