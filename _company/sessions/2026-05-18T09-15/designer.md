## 💻 Maru Company Landing Page Mockup: System Integrity Check
**목표:** 잠재 고객의 미인식 기회 손실($L_{Occ}$)을 정량화하여 위기감을 조성하고, 당사 솔루션이 유일한 '시스템 무결성 보증'임을 확립한다.

### 🖼️ [Wireframe Structure]
전체 페이지는 스크롤에 따라 4개의 명확히 분리된 **섹션(System Block)**으로 구성됩니다. 각 섹션의 전환은 애니메이션을 통해 다음 단계로의 '시스템적 상태 변화'를 경험하게 합니다.

---

### 🔴 Section 1: 시스템 경고 대시보드 (The Crisis Hook)
**목표:** 스크롤 진입과 동시에 충격적인 $L_{Occ}$ 수치를 노출하여 주의 집중(Attention Capture).
**디자인 특징:** 실제 운영되는 엔지니어링 대시보드처럼 보이게 구성.

| 요소 | 내용/규칙 | 개발 지침 (Interaction) |
| :--- | :--- | :--- |
| **헤드라인** | **[SYSTEM ALERT: CRITICAL] 시스템 무결성 손상 감지.**<br>당신의 비즈니스 로직은 현재 미인식 기회 손실($L_{Occ}$) 상태에 있습니다. | `<h1>` 태그는 고정된 폰트(Monospace)와 Critical Red로 강조. 페이지 진입 시, 타이핑 애니메이션처럼 글자가 순차적으로 나타나야 함. |
| **$L_{Occ}$ 위젯** | **[Loss Detection Widget]** - 실시간 계산기 형태의 대시보드 위젯. 3개의 입력값(A, B, C)을 사용자에게 제시하고, 이들이 변수화되면서 $L_{Occ} = A \times B \times C$를 산출하는 과정을 애니메이션으로 보여줘야 함. | **핵심 애니메이션:** 페이지가 로드되면 위젯의 입력값 필드가 비어있다가(0) 마치 '실시간 데이터 스트림'처럼 값이 폭발적으로 증가하며 최종 $L_{Occ}$ 값에 도달해야 함. (이 과정 자체가 충격). |
| **Critical Red 경고** | **[PROTOCOL VIOLATION DETECTED]**: 현재 운영 방식은 비효율적입니다. 이대로 두면 재무적 손실 위험도가 임계치(Threshold)를 초과합니다. | 배경에 미세한 노이즈 필터 효과와 함께, 위젯 주변으로 붉은색의 경고 패턴(Pulse/Flicker)을 오버레이 해야 함. (Red Alert #1). |
| **CTA** | *[Analyze Vulnerability Protocol]* → 다음 단계로 진행 버튼. |

### 🟡 Section 2: 시스템 상태 전이 오류 분석 (The Technical Diagnosis)
**목표:** 문제가 단순한 '비효율'이 아니라 '시스템 구조적 결함'임을 기술적으로 증명하여 권위를 확보.
**디자인 특징:** 복잡한 플로우차트 및 로그 시각화.

| 요소 | 내용/규칙 | 개발 지침 (Interaction) |
| :--- | :--- | :--- |
| **헤드라인** | **[ERROR LOG] 시스템 상태 전이(State Transition) 오류 발생 원인 분석.**<br>당신의 워크플로우는 'A $\to$ B'라는 선형 구조를 따르지 못하고, 예상치 못한 무한 루프 또는 누락된 Edge로 인해 데이터가 손실됩니다. | 전체 섹션 배경을 다이어그램 뷰어처럼 설계 (좌측에 노드, 우측에 엣지). |
| **다이어그램** | 현재 고객의 비즈니스 플로우를 간소화된 상태 전이 그래프로 보여줌. 핵심 오류 지점(Missing Edge)은 회색으로 처리하고, 그곳에서 발생하는 데이터 손실 경로를 화살표와 경고 문구로 강조. | **애니메이션:** 사용자가 스크롤을 내리면, 플로우차트의 엣지들이 마치 전력 흐름처럼 빛나며 지나가다가, 오류 지점(Missing Edge)에 도달할 때 갑자기 전류가 끊기거나 폭발하는 시각적 효과를 연출해야 함. |
| **경고 컴포넌트** | **[WARNING: STALE DATA]**: 이 데이터는 시스템 구조적으로 접근 불가능하거나, 이미 유효성을 상실한(Stale) 데이터를 기반으로 합니다. 수정이 필요합니다. | 경고 메시지 박스에 앰버/골드 색상 코드를 적용하고, 깜빡이는 커서 효과를 주어 '현재 작업 중인 문제'임을 강조 (Red Alert #2). |

### 🟢 Section 3: Maru Protocol 소개 (The Solution Integration)
**목표:** 문제를 진단한 후, 당사의 솔루션을 유일무이한 '패치(Patch)' 또는 '프로토콜 업데이트'로 포지셔닝.
**디자인 특징:** Clean하고 구조적인 인터페이스 전환.

| 요소 | 내용/규칙 | 개발 지침 (Interaction) |
| :--- | :--- | :--- |
| **전환 문구** | **[PROTOCOL UPDATE REQUIRED]**: 시스템 무결성 확보를 위해 'Maru Protocol'을 통합해야 합니다. | 섹션 전환 시, 배경에 `[PATCHING...]` 또는 `[INITIALIZING NEW PROTOCOL]` 같은 기술적 텍스트가 빠르게 스크롤되거나 깜빡여야 함. (State Change 시각화). |
| **핵심 기능 설명** | 복잡한 개념은 단순 나열하지 않고, 'Before'와 'After'의 상태 전이 비교를 통해 설명한다. 예: Before(파편적 데이터) $\to$ After(완전 통합된 그래프). | 비주얼 자료는 3D 모델링 또는 인터랙티브 컴포넌트를 활용하여, 사용자가 마우스 오버 시 내부 구조가 '정렬'되는 느낌을 주어야 함. |
| **신뢰 요소** | 개발자 중심의 용어(e.g., Atomic Transaction, Local-first Architecture)를 사용하여 엔지니어적 신뢰도를 극대화한다. |

### 🔴 Section 4: 최종 보증 및 CTA (The Conversion Guarantee)
**목표:** 망설이는 고객에게 마지막 기술적 충격과 행동 유도(CTA)를 제공.
**디자인 특징:** 가장 강렬하고 단순하며, 즉각적인 액션을 요구하는 레이아웃.

| 요소 | 내용/규칙 | 개발 지침 (Interaction) |
| :--- | :--- | :--- |
| **최종 위기 경고** | **[FINAL WARNING: $L_{Occ}$ Accumulation Rate is Exponential]**: 이대로 방치할 경우, 손실액은 기하급수적으로 증가합니다. 지금 바로 행동해야 합니다. | 배경을 다시 Critical Red 톤으로 전환하고, 시계 카운트다운 타이머(가상)를 배치하여 시간적 압박감을 부여 (Red Alert #3). |
| **CTA 영역** | **[INITIATE SYSTEM PATCH]**: 가장 강력한 CTA 버튼. '시작하기' 같은 일반적인 문구 금지. 반드시 '시스템 무결성 진단 시작' 또는 '손실액($L_{Occ}$) 감소 보증 받기' 등의 전문 용어를 사용해야 함. | 버튼은 물리적으로 크고, 주변에 빛을 발하는 듯한 Glow 효과를 주어 시각적 무게감을 극대화. |

### 🎨 Color Palette
| Name | Hex Code | Usage Context | Notes |

| **Critical Red** | `#DC2E2E` | 🚨 **최대 경고**, $L_{Occ}$ 폭발 시, Protocol Violation. | 충격적이며 즉각적인 주의 요구. 애니메이션 필수. |
| **Warning Amber** | `#FFC107` | ⚠️ 잠재 위험 지점, 데이터 불완전성 (Missing Edge). | 주의 환기 및 다음 단계를 암시. |
| **Success/Info Blue** | `#2196F3` | 정상 작동 상태, 'Maru Protocol' 도입 완료 후의 안정화. | 신뢰와 해결책 제시. |

### 🎨 Typography & Component Standards
*   **Font Family:** Inter 또는 MonoLisa (Monospace 계열) 사용을 원칙으로 합니다. 전문적이고 코딩하는 듯한 느낌을 유지해야 합니다.
*   **Component: Alert Box:** 모든 경고 메시지는 일반 박스가 아닌, `[SYSTEM LOG]`나 `<<< PROTOCOL ALERT >>>`와 같은 시스템 로그 스타일의 테두리와 배경색(Red/Amber)을 가져야 함.
*   **Animation Rule (State Change):** 섹션 간 전환 시 반드시 0.5초 이상의 **로딩 인디케이터(Loading Indicator)**를 삽입해야 합니다. 이는 사용자가 '시스템이 다음 단계를 처리 중'이라는 착각을 일으켜 신뢰도를 높이는 핵심 장치입니다.
