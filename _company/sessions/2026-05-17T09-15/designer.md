## 📄 Missing Edge Shock Component Design Spec V5.0
**목표:** 사용자에게 데이터 파편화로 인한 재무적 손실액($Loss Amount$)을 즉각적으로 인지시키고, 통합 인덱싱 엔진을 통한 잠재 추가 순이익($PVU$) 기회 발견으로 유도한다.
**최종 사용처:** 랜딩 페이지 (Hero/Feature Section)
**필수 전제 조건:** 스크롤 위치(Scroll Position) 및 가상 데이터 API 호출 시점을 핵심 트리거로 사용함.

### 1. 컴포넌트 개요 및 상태 흐름 다이어그램

| 단계 | 상태 명칭 (State Name) | 사용자 경험 (UX Goal) | 감정적 목표 (Emotional Trigger) | 주요 색상 코드 |
| :---: | :---: | :---: | :---: | :---: |
| **1** | **Fragmentation View** (기본/파편화 인식) | 현재 데이터의 사일로화된 상태를 인식하게 한다. | *무관심 $\to$ 미세한 불편함* | Neutral Gray (#D3D3D3), Accent Blue (Primary Brand Color) |
| **2** | **Crisis Shock View** (위기 충격 자각) | 데이터 결핍이 곧 재무적 손실임을 수치적으로 쇼크 받게 한다. | *불안 $\to$ 위협감* | Warning Red (#E74C3C), Amber Gold (#F39C12) |
| **3** | **Opportunity View** (기회 발견/해결책 제시) | 통합 인덱싱 엔진을 통한 잠재적 수익($PVU$) 기회를 명확히 확인한다. | *안도감 $\to$ 강력한 기대* | Success Gold (#FFD700), Deep Blue (Trust Color) |

### 2. 단계별 와이어프레임 및 인터랙션 로직 상세 정의

#### [Stage 1: Fragmentation View]
**Trigger:** 페이지 진입 (Initial Load / Scroll Y=0px ~ Y=400px)
**레이아웃:** 가로 방향으로 분리된(Disconnected) 데이터 카드 3~5개 배치. 각 카드는 서로 물리적으로 떨어져 있음.

| 요소 | 디테일 스펙 | 인터랙션 로직 |
| :--- | :--- | :--- |
| **데이터 카드** | 개별 제목, 메타데이터 등 표준 정보 포함. 모든 카드가 동일한 배경색(Gray)에 독립적으로 존재함. | 사용자가 마우스 커서를 각 카드 위로 가져갈 때(Hover), 해당 카드의 경계선 주변으로 미세한 떨림 효과(Subtle Pulse)만 발생하고, 다른 카드에는 아무런 반응이 없음. (분리됨 강조). |
| **헤드라인** | "데이터는 모여있지만 연결되어 있지 않습니다." 와 같은 메시지 배치. | 스크롤 다운을 유도하는 형태로, 텍스트가 마치 파편처럼 분산된 느낌으로 나타나야 함. |

#### [Stage 2: Crisis Shock View]
**Trigger:** 스크롤 위치 도달 (Scroll Y=400px ~ Y=650px) — **핵심 전환 지점**
**변화 로직:** Stage 1의 모든 데이터 카드가 강렬한 시각적 충격과 함께 중앙으로 수렴하며 'Missing Edge'가 발생하는 순간을 연출한다.

| 요소 | 디테일 스펙 | 애니메이션/인터랙션 (Timing Critical) |
| :--- | :--- | :--- |
| **전환 효과** | 모든 데이터 카드가 폭발하듯(Explosion effect) 중앙의 비어있는 공간으로 빨려 들어가는 듯한 시각적 연출. 배경에 노이즈/글리치(Glitch) 필터를 순간적으로 적용. | **0ms:** (Stage 1 $\to$ Stage 2 시작). 모든 요소가 강렬하게 흔들림 (Shake effect).<br>**500ms:** 가장 큰 충격파(Shockwave) 발생. 배경의 글리치 효과 최대화.<br>**800ms:** 중앙에 'Missing Edge' 경고창 컴포넌트가 붉은색으로 급작스럽게 등장하며 모든 시선 집중. |
| **경고창** | 크기: 전체 화면 폭의 90%. 배경색: Dark Red (#A94442). 제목: "🚨 [WARNING] 데이터 사일로로 인한 재무적 손실 경고!"<br>핵심 메시지: $\text{Potential Loss Amount: } \mathbf{\$X,XXX.XX}$ (Red/Amber 색상 강조). | **0ms:** 텍스트가 타이핑되듯 빠르게(Typewriter effect) 나타나며 충격적인 수치($Loss Amount$)를 보여줌.<br>**300ms:** 경고창 주변에 불규칙적이고 빠른 깜빡임 효과(Flicker/Pulse)를 지속적으로 적용하여 위기감을 극대화. |
| **CTA (Shock)** | "손실액 확인 및 연결 엔진 보기" 버튼. 색상: 강렬한 Amber Gold (#FFC107). | 사용자가 이 버튼을 클릭할 때, 페이지가 Stage 3으로 즉시 부드럽게 스크롤 이동해야 함 (Smooth Scroll to next section). |

#### [Stage 3: Opportunity View]
**Trigger:** CTA 클릭 또는 스크롤 위치 도달 (Scroll Y=700px ~ 끝)
**레이아웃:** 데이터 사일로를 연결하는 추상적 인덱스 그래프(Graph Visualization)가 중앙에 배치됨.

| 요소 | 디테일 스펙 | 애니메이션/인터랙션 로직 |
| :--- | :--- | :--- |
| **그래프 시각화** | 초기에는 데이터 노드들이 분산되어 있다가, 마치 자석처럼 서로 끌어당겨지며 선(Edge)이 생성되는 과정을 보여준다. | **0ms:** (Stage 2 $\to$ Stage 3 전환). 어둠 속에서 낮은 주파수의 전기적 '윙-' 하는 소리와 함께 그래프 노드들이 점멸하며 나타남.<br>**1000ms:** 가장 중요한 Edge가 발견되면서(Connecting two previously separate nodes), 해당 Edge를 따라 황금빛 빛줄기(Golden Arc)가 빠르게 이동함. |
| **PVU 계산 컴포넌트** | '연결된 데이터'가 실제 돈으로 변환되는 과정을 시각화하는 인터랙티브 카운터/계산기 영역 배치. | 사용자가 "분석 시작" 버튼을 누르면, 그래프 상의 연결 선들이 활성화되며 백그라운드에서 **PVU 계산 로직**이 돌아가는 듯한 애니메이션(Loading spinner + data flow visual)을 보여줌.<br>**결과 출력:** 최종적으로 $\text{Potential Value Unit (PVU): } \mathbf{\$XX,XXX.XX}$ 가 큼지막하고 신뢰감 있는 Deep Blue 박스 안에 나타남. |
| **CTA (Solution)** | "지금 바로 통합 인덱싱 엔진 사용하기" 버튼. 색상: Deep Blue (#007BFF). | 이 CTA는 최종 전환 지점이며, 클릭 시 서비스 온보딩/가입 페이지로 이동해야 함. 전 단계의 모든 위기감과 충격을 해소하는 '안정감'을 주어야 함. |

### 3. 기술적 구현 및 검토 사항 (Dev Notes)

1.  **스크롤 기반 트리거링:** Stage 2와 Stage 3의 전환은 단순 섹션 분리가 아닌, 스크롤 위치에 따라 **컴포넌트가 변형(Morphing)**되거나 **애니메이션이 진행되는 과정**으로 인식되어야 합니다. (예: CSS `transform` 및 JS `IntersectionObserver` 활용).
2.  **사운드 디자인:** 각 단계 전환 시점마다 사운드를 반드시 삽입해야 함.
    *   Stage 1 $\to$ Stage 2: 낮은 주파수의 불안정한 노이즈 + 경고음 (Alarm Tone)
    *   Stage 2 $\to$ Stage 3: 고주파의 밝은 전자음(Synth rising tone) + 해결된 느낌의 '딩' 소리.
3.  **반응형 디자인:** 모든 컴포넌트는 모바일 환경에서 가장 충격적인 순간이 시각적으로 잘 전달되도록 (특히, 손실액 폰트 크기 및 배치) 조정되어야 합니다.
