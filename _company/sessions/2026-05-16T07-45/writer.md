### 💡 Idea #1: 데이터 사일로 파괴 - '시간 비용' 절감에 초점

**[핵심 가치 전달]**: 시간과 노력을 돈으로 환산하여 보여주기.
**[후킹 제목/Hook]**: **"노션 → 블로그까지, 이 과정을 하루에 몇 시간 낭비하고 계십니까?"** (직관적이고 구체적인 질문)

| 요소 | 내용 (화면 비주얼 가이드) | 내레이션/카피라이팅 (엔지니어톤) |
| :--- | :--- | :--- |
| **HOOK / Problem (0~5초)** | (빠르게 전환되는 앱 아이콘들: Notion, Slack, Google Docs 등) 사용자가 마우스를 여러 툴 사이에서 왔다 갔다 하는 모습. (좌절하는 표정의 사용자 클로즈업) | "하나의 자료를 만들고 끝이 아닙니다. 노션에 정리된 인사이트를 슬랙 팀원에게 공유하고... 이걸 다시 블로그 포스팅으로 옮기려면, **반복적인 복사-붙여넣기**가 필수입니다." |
| **Frustration / Gap (5~15초)** | 화면에 '데이터 연결 실패', '수동 작업 3시간' 등의 경고 문구가 빨간색으로 뜨면서 시스템 오류처럼 연출. | "이게 바로 1인 기업가가 직면하는 가장 큰 구조적 문제입니다. 자료는 이미 존재하는데, 이 **A와 B를 논리적으로 연결해 줄 워크플로우 자체가 부재**한 거죠." |

| **마무리 CTA (30초 이후)** | 제품의 핵심 기능 시연 화면. "자동 연결된 워크플로우" 강조. | "**데이터 사일로를 구조적으로 파괴하고, 당신의 시간을 100% 아끼세요.** Maru Company가 진짜 '회사'처럼 일하게 만듭니다." |

### 💡 Idea #2: 잠재적 기회 포착 - '수익 예측(PVU)'에 초점
**[타겟 Pain Point]**: 비선형적 지식 구조화 어려움 (어떤 지식이 다음 사업 기회로 연결될지 모름)
**[핵심 가치 전달]**: 손실 방지를 넘어, **새로운 수익의 가능성(PVU)**을 예측해줌.
**[후킹 제목/Hook]**: **"당신의 '노하우' 중, 아무도 모르는 30%짜리 잠재 수익이 있습니다."** (구체적인 수치를 제시하며 호기심 자극)

**[스크립트 초안]**

| 요소 | 내용 (화면 비주얼 가이드) | 내레이션/카피라이팅 (엔지니어톤) |
| :--- | :--- | :--- |
| **HOOK / Problem (0~5초)** | 사용자가 메모한 지식 노트들(개인 파일, PDF 스크랩 등)이 산더미처럼 쌓여있는 이미지. 마치 미로 같은 느낌 연출. | "당신은 수많은 자료를 모았습니다. 하지만 이 정보들이 어떻게 연결되는지, 어떤 결핍을 채우고 있는지 아십니까? 그저 '지식'으로만 머물러 있어요." |
| **Frustration / Gap (5~15초)** | 지식 노드(점)는 많은데, 선(Edge)이 끊겨있는 다이어그램 애니메이션. 경고 메시지: "Missing Edge Detected." | "지금까지의 KMS는 '무엇을 아는지'만 보여줬습니다. 하지만 진짜 필요한 건, **A라는 지식이 B라는 기회와 연결되어 발생할 수 있는 잠재적 가치**를 예측하는 겁니다." (기술 용어 사용) |
| **Solution / CTA (15~30초)** | Maru Company UI가 데이터 노드 사이의 끊어진 선(Edge)을 찾아내고, 그 위에 'Potential Value Unit'이라는 골드 하이라이트를 입히는 애니메이션. | "Maru Company의 Missing Edge 분석은 단순한 연결점을 찾는 것이 아닙니다. 이 결핍 지점에서 **얼마나 큰 잠재적 순이익(PVU)**이 발생할 수 있을지, 데이터를 기반으로 정량화하여 제시합니다." (기술과 비즈니스의 결합 강조) |
| **마무리 CTA (30초 이후)** | 예측된 PVU가 그래프로 상승하는 모습. 명확한 행동 유도 버튼 강조. | "**당신의 지식을 그냥 '보관'하지 마십시오. 잠재적 자산으로 '활용'하세요.** Maru Company와 함께, 다음 수익 기회를 발견하세요." |

### 💡 Idea #3: 콘텐츠 확장성 - '노동력 최소화'에 초점
**[타겟 Pain Point]**: 콘텐츠의 원천 자산 활용 극대화 불가 (인터뷰 녹취록 하나로 여러 포맷을 만들고 싶은데 수작업이 너무 많음)
**[핵심 가치 전달]**: 1개의 투입(Input)으로 N개의 아웃풋(Output) 생성.
**[후킹 제목/Hook]**: **"녹취록 파일 하나가, 블로그 포스팅 + 숏폼 스크립트 + 강의 목차로 변하는 마법."** (구체적인 결과물 목록 제시)

**[스크립트 초안]**

| 요소 | 내용 (화면 비주얼 가이드) | 내레이션/카피라이팅 (엔지니어톤) |
| :--- | :--- | :--- |
| **HOOK / Problem (0~5초)** | 마이크 녹음기 또는 인터뷰 현장의 분위기. 고품질의 원본 데이터(녹취록)를 강조. | "시간과 에너지를 들여 얻은 단 하나의 소중한 자산, 바로 '콘텐츠'입니다. 하지만 이 녹취록을 그대로 쓰긴 어렵죠." |
| **Frustration / Gap (5~15초)** | 사용자가 텍스트 편집기에서 내용을 잘라내고(Ctrl+C), 포맷에 맞게 다듬는 수많은 과정이 시간 흐름과 함께 지연되는 연출. | "블로그 문체로 바꾸려면? → 제목 뽑기. 숏폼 스크립트로 만들려면? → 임팩트 있는 컷 분할. 이 과정이 반복됩니다. **콘텐츠를 '확장'하는 것이 아니라, 콘텐츠에 노동력을 '덧붙이는' 작업**입니다." |
| **Solution / CTA (15~30초)** | Maru Company AI가 녹취록을 입력받자마자, 분석 결과를 바탕으로 ① 블로그 SEO 클러스터 구조 ② 숏폼의 대본 포맷 ③ 강의 아웃라인 개요 등 3가지 다른 형식의 결과물이 동시에 완성되는 시각화. | "Maru Company는 원천 데이터를 한 번만 처리합니다. AI 에이전트가 **내용의 핵심 논리(Schema)**를 파악하여, 타겟 플랫폼과 목적에 맞는 최적의 아웃풋 포맷으로 자동 변환해주는 겁니다." (기술 용어 사용) |
| **마무리 CTA (30초 이후)** | 'Input 1개' $\rightarrow$ 'Output N개'가 명확한 그래프로 제시됨. | "**더 이상 콘텐츠에 노동력을 붙이지 마십시오.** 원천 데이터의 가치를 최대치로 끌어올리는 자동화, Maru Company와 함께하세요." |
