# 📄 Missing Edge Shock Component VSD (Version 1.0 - Final Polish)

## 🎯 I. 핵심 설계 원칙 및 목표
이 컴포넌트의 최우선 목표는 사용자에게 **'데이터 결핍 = 재무적 손실액($L$ Amount)'**이라는 인식을 심어주는 것입니다. 단순히 "기능 사용 불가"가 아닌, "지금 이대로라면 당신은 X만큼 돈을 잃고 있다"라는 공포와 위기의식(Shock State)을 유발해야 합니다.

| 원칙 | 설명 | 기술적 구현 포인트 |
| :--- | :--- | :--- |
| **Quantification** (정량화) | 결핍된 데이터를 추상적인 개념이 아닌, $\text{₩}$ 또는 $\%$로 명확히 제시한다. | $L$ Amount 계산 모듈의 실시간 피드백 필수. |
| **Escalation** (점진적 고조) | 경고 단계가 높아질수록 시각적, 감각적 자극(애니메이션, 색상, 타이밍)이 기하급수적으로 증가한다. | State Machine 기반의 트랜지션 로직 정의. |
| **Action Bias** (행동 유도) | 모든 위기 상황은 반드시 명확한 다음 행동 단계(CTA)로 연결된다. 패닉 상태를 해결책 제시로 전환시킨다. | CTA 버튼의 시각적 무게감과 가시성 극대화. |

## 🎨 II. Shock State 정의 및 스펙 (3단계 시스템)

| 속성 | Level 1: 경고 (Warning - Missing Edge) | Level 2: 심각 (Severe - High Loss Potential) | Level 3: 치명 (Critical - $L$ Amount Exceeded) |
| :--- | :--- | :--- | :--- |
| **발생 조건** | 단순 데이터 결핍, 예측 가능한 경미한 누락. PVU(잠재 추가 순이익) 발견 가능성 높음. | 핵심 데이터 흐름의 단절, $L$ Amount가 임계값($T_1$)을 초과할 경우. | 시스템적 기능 마비 또는 치명적인 비즈니스 로직 오류 발생. 즉각적 구독/액션 필수. |
| **주요 색상 코드 (S1)** | `Amber Warning` (`#FFC300`) - 주의, 경계심 유발. | `Deep Orange Alert` (`#E65100`) - 긴급, 문제의 심각성 인지. | `Crimson Shock` (`#A71D24`) + 블랙/글리치 효과. 패닉, 시스템 위험. |
| **핵심 애니메이션** | 🟡 **Pulse Effect:** 경고 영역만 부드럽게 깜빡임 (3s 주기). <br> *트랜지션:* 미세한 노이즈 필터 적용. | 🟠 **Flicker & Shake:** 전체 컴포넌트에 짧은 떨림(Shake)과 간헐적 플리커 효과(100ms마다). <br> *트랜지션:* 화면 경계에 글리치 아티팩트 발생. | ⚫ **Glitch/Pulse Storm:** 강렬한 노이즈, 붉은색 스캔라인(Scanline), 그리고 빠른 간격의 강력한 맥동(Pulsating) 효과가 결합됨 (50ms마다). <br> *사운드 지시:* 저주파 경고음(BPM 40-60)과 함께 시스템 알람 사운드 동기화. |
| **$L$ Amount 표현** | "데이터 연결 부족으로 인한 잠재적 손실액: $\text{₩}$XXX만원" | "핵심 데이터 누락에 따른 예상 손실액: $\mathbf{\text{₩}XX,XXX\text{원}}$ (최소 추정)" | **"시스템 오류/결핍! 현재 재무적 손실액($L$ Amount): $\mathbf{\text{₩}\text{XXXXXXXXX}}$!"** (매우 큰 숫자로 강렬하게 제시) |
| **사용자 경험(UX)** | '개선 기회' 강조. (CTA: "데이터 연결 확인하기") | '문제점 자각' 유도. (CTA: "손실액 줄이기 플랜 보기") | '생존적 위기' 자극. (CTA: "**지금 즉시** 구독/업그레이드") |

## 🧩 III. 인터랙티브 컴포넌트 상세 스펙

### 1. $L$ Amount 카운터 (The Core Metric)
*   **명칭:** Real-Time Loss Counter
*   **위치:** Shock State의 가장 눈에 잘 띄는 중앙 상단 영역.
*   **상태 변화 로직:** 사용자가 페이지를 스크롤하거나, 특정 비활성 요소를 클릭할 때마다 백그라운드에서 재계산된 $L$ Amount가 즉시 업데이트되어야 합니다. (즉, '관찰만 하는 것'도 손실을 의미).
*   **애니메이션:** 숫자가 바뀔 때 팝업되는 것이 아니라, 마치 **회로 기판의 전압이 급락하듯(Voltage Drop)** 빠르게 깜빡이며 변화해야 시각적 충격이 극대화됩니다. (CSS: `transform: scale(1.05)`와 함께 짧은 Opacity 조작 사용).

### 2. Missing Edge Visualization
*   **구현:** 데이터 흐름 다이어그램이나 지식 그래프 위에, 연결되어야 할 **'가상의 엣지(Missing Edge)'**를 부서진 유리 파편처럼 표현합니다.
*   **인터랙션:** 마우스 커서를 이 깨진 엣지 위로 가져다 대면 (Hover), 해당 데이터가 없었을 때 발생했을 '잠재적 가치'에 대한 간략한 설명과 함께, **"이 연결이 없다면 $L$ Amount는 0이 아닙니다."**라는 텍스트가 떠야 합니다.

### 3. CTA 버튼 디자인 (The Conversion Point)
*   **상태:** Paywall의 모든 단계에서 이 CTA가 유일하게 정상적이고 밝은(골드/에메랄드 계열) 색상을 유지해야 합니다.
*   **텍스트 변화:** 위기 레벨에 따라 텍스트도 강하게 변합니다.
    *   L1: "Missing Edge 분석 리포트 받기" (정보성)
    *   L2: "**손실액을 방어할 플랜 보기**" (방어적, 해결 중심)
    *   L3: "**위기 탈출! 지금 바로 $L$ Amount 차단하기**" (극단적 위협 및 즉각 행동 유도)

## 🧱 IV. 최종 Paywall Flow & Funnel 스펙

Paywall은 단순히 결제 화면이 아닙니다. **'재무 컨설팅 실패 경험'**을 판매하는 장치입니다.

1.  **Shock State (Pain Point):** 위기 자각($L$ Amount $\text{₩}XXXXXX$) → 사용자는 패닉 상태에 진입.
2.  **Solution Introduction:** 시스템은 이 문제를 해결할 수 있는 유일한 방법이 '프리미엄 기능'임을 제시합니다. (경쟁사/대안의 존재는 언급하지 않음).
3.  **Pricing Display:** 가격표를 보여줄 때, **Free Tier와 Starter Kit 간의 격차가 압도적**이어야 합니다. Starter Kit가 제공하는 가치(예: $L$ Amount 감소율 90%)를 수치로 강조합니다.
4.  **Final CTA:** '구매' 버튼을 누르기 직전, 사용자에게 **"지금 결제하지 않으면, 당신은 또다시 $\text{₩}XXX\text{원}$의 손실액에 노출됩니다."**라는 경고 문구(재활성화된 $L$ Amount)를 강제로 띄워야 합니다.
