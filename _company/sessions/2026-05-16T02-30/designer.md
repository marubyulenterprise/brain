# 🔗 지식 그래프 시각화 시스템 디자인 브리프 (V1.0)
**Project:** Maru Knowledge Graph Visualization System
**Goal:** 데이터의 논리적 연결고리(Edge Discovery)를 직관적이고 신뢰감 있게 전달하는 비주얼 원칙 확립.
**톤앤매너:** 기술적, 정밀함, 명료함 (Technical, Precise, Clear).

## 1. 컬러 팔레트 정의 (Color Palette Definition)

지식 그래프의 시각화는 배경과 정보 요소 간의 대비가 핵심입니다. 너무 많은 색상을 사용하기보다, *구분자(Differentiator)*로서의 색상에 집중합니다.

| 역할 | 이름 | Hex Code | 사용 원칙 |
| :--- | :--- | :--- | :--- |
| **Primary Background** (배경) | Data Canvas | `#0A192F` | 깊은 밤하늘 같은 네이비/블랙 계열. 데이터가 떠오르는 느낌을 주며, 다른 색상(정보)의 대비 효과 극대화. |
| **Secondary Text** (텍스트) | UI White | `#E6F1FF` | 배경 위에서 가독성이 높은 밝은 흰색. 기본 텍스트 및 레이블에 사용. |
| **Accent - Focus** (강조/흐름) | Discovery Cyan | `#4D9DE0` | **가장 중요한 색상.** 활성화된 Edge, 핵심 노드, 사용자 액션 플로우에만 사용. '발견'의 순간을 상징. |
| **Neutral - Concept Node** (개념 노드) | Teal Structure | `#5A9E8C` | 추상적인 개념(Concept)이나 이론적 틀을 나타내는 기본 색상. 안정감과 체계성을 부여. |
| **Neutral - Entity Node** (주체 노드) | Slate Blue | `#6B7D92` | 사람, 회사 등 실재하는 주체(Person/Entity)를 나타내어 현실감을 부여. |
| **Accent - Error/Warning** (경고) | Muted Red | `#C0534E` | 연결 실패, 누락된 정보 등 시스템적 오류나 주의가 필요한 경우에만 사용. |

## 2. 타이포그래피 및 아이콘 시스템 (Typography & Icon Set)

### A. 타이포그래피 (Typography)
*   **폰트:** **Pretendard** (혹은 Inter).
*   **규칙:** 모든 정보는 굵직한 헤드라인(H1, H2)과 설명 텍스트 간의 크기 대비를 명확히 하여 계층 구조(Hierarchy)를 보여줘야 합니다. 그래프 내부의 레이블링은 최대 2줄을 넘지 않도록 엄격하게 제한합니다.
*   **활용:** 데이터 값이 가장 중요하므로, 숫자는 항상 시스템 기본 서체보다 약간 더 크고 무게감 있게 처리하여 시선이 먼저 가도록 유도합니다.

### B. 아이콘 세트 (Icon Set)
*   **스타일:** Line-based / Minimal SVG Glyphs. (채워진 도형보다는 선으로만 이루어진 간결한 형태).
*   **목적:** 노드 유형을 시각적으로 즉시 식별하게 합니다.
    *   `Concept Node`: 💡 (전구 모양) 또는 `✨` (반짝임 기호)를 활용하여 추상성을 강조.
    *   `Person Node`: 👤 (사람 실루엣) 아이콘을 기본으로 사용.
    *   `Relationship Edge`: 화살표(Arrowhead) 아이콘 그 자체를 디자인 요소로 승화시켜 방향성을 부여.

## 3. 핵심 컴포넌트 가이드라인

### A. 노드 (Node: Concept, Person)
노드는 단순한 원이 아닌, 정보를 담는 '데이터 컨테이너'처럼 보이게 설계합니다.

*   **형태:** 모서리가 부드럽게 처리된 직사각형 또는 육각형(Hexagon). (직선적이지만 지나치게 딱딱하지 않은 느낌).
*   **배경/색상:** 섹션 1의 지정 색상 사용.
*   **텍스트 배치:** 노드의 중심에 주요 키워드를 배치하고, 필요시 하단 모서리에 **Node Type Tag**를 작게 삽입합니다 (예: `[CONCEPT]`, `[PERSON]`).

### B. 엣지 (Edge: 관계의 흐름)
이 부분이 제품의 핵심 가치를 담는 부분이며, 가장 많은 공을 들여야 합니다. 단순한 선(Line)이 아닙니다.

*   **형태:** **방향성 있는 점선 또는 파동 형태의 경로**. (Wavy/Dashed Line with Arrowhead).
*   **색상:** 기본은 `#4D9DE0` (Discovery Cyan)을 사용하되, 데이터가 흐르는 것처럼 가장자리만 미세하게 빛나는(Glow Effect) 그라데이션 효과를 적용합니다.
*   **직관성 강화:** 엣지 위에 마우스 오버 시, 연결된 노드 두 개 사이의 관계 정의(`[관계 타입: X는 Y에 영향을 준다]`)가 작은 툴팁으로 나타나야 합니다.

## 4. 데이터 흐름 애니메이션 가이드 (Animation Guideline)

애니메이션은 '마법'이 아니라 **'계산 과정'**처럼 보이게 해야 합니다. 빠르지만, 논리적입니다.

1.  **입력 단계 (Input Phase):** 사용자가 자료를 넣으면, 시스템이 즉시 노드를 생성하는 것이 아닙니다. 먼저 입력된 데이터 파편들이 흩뿌려지며(Particles), 작은 점들로 변환됩니다.
2.  **처리 단계 (Processing/Computation Phase):** 화면 전체에 미세하게 깜빡이는 그리드 패턴이나 '계산 중'임을 암시하는 저주파의 빛이 지나갑니다. (기술적 신뢰감 부여). 이 과정은 1초를 넘지 않습니다.
3.  **발견 단계 (Discovery/Output Phase):** 처리 과정을 거치자마자, 가장 중요한 노드와 노드 사이의 관계(Edge)가 **강렬한 Cyan Glow**를 내며 빠르게 연결되는 애니메이션을 보여줍니다. 마치 '논리적 트랜잭션이 완료되었다'는 느낌을 주는 것이 목표입니다.
