### 📄 [최종 애니메이션 자산 통합 브리프] - Loss Funnel Flow (v1.0)

**목표:** 사용자가 데이터의 '미흡함'을 기술적 결함(Glitch)으로 인지하고, 이를 '측정 가능한 재무적 기회비용 손실($X,XXX)'으로 전환하여 해결책에 강하게 몰입하도록 유도한다.
**사용 시점:** 사용자가 현재 연결된 Edge 개수가 시스템이 권장하는 임계치(예: 5개)를 초과하지 못하거나, 데이터 간의 'Missing Edge'가 발견되었을 때.

| 단계 | 사용자 경험 (Emotion) | 시각 효과 (Visual Effect) | 색상 코드 / 의미 |
| :--- | :--- | :--- | :--- |
| **Pain Point** $\rightarrow$ | **당혹감, 불완전함.** ("뭔가 빠진 것 같은데?") | 일반적인 노이즈, 흐릿한 경계. (Ambient Noise) | **앰버/그레이 (#FFC107 / #AAAAAA):** *경고.* 임계치 근접. 데이터의 '미흡'함을 알림. |

### 🛠️ 2. 스테이지별 인터랙티브 컴포넌트 스펙 (Lottie/SVG Specification)

#### A. [STAGE 1] Pain Point Detection (The Hint - 앰버 단계)
*   **발동 조건:** Missing Edge가 발견되었으나, 아직 심각한 손실은 아닐 때.
*   **비주얼 목표:** 사용자가 '무엇인가 부족하다'는 감지적 불편함을 느끼게 한다.
*   **애니메이션 스펙 (Lottie):**
    1.  화면 중앙의 특정 Edge(Missing Edge) 주변으로 앰버 색상의 **잔물결(Ripple Effect)**이 퍼진다.
    2.  노드와 노드 사이의 경계선에 간헐적인 **'Pulse & Flicker' 애니메이션**을 적용하여, 데이터 흐름이 불안정하다는 인상을 준다.
    3.  *오디오 힌트:* 낮고 반복적인 '삐-' 하는 전기적 잡음(Humming Noise) 사운드를 삽입한다.

#### B. [STAGE 2] Glitch Loss Crisis (The Threat - 레드 단계)
*   **발동 조건:** Missing Edge가 발견된 개수가 임계치를 초과하여, 잠재적인 기회비용 손실이 발생했을 때. **(핵심)**
*   **비주얼 목표:** 시스템 자체가 실패하는 듯한 공포와 위협을 극대화하고, 그 피해액($X,XXX)에 집중시킨다.
*   **애니메이션 스펙 (Lottie/SVG):**
    1.  **[Glitch Effect]:** 화면 전체가 찰나의 순간 동안 수평으로 **'스크린 티어링(Screen Tearing)'**되며 왜곡된다. 이는 디지털 시스템 자체의 오류로 느껴지게 한다.
    2.  **[Broken Edge]:** 연결되어야 할 가장자리(Edge)들이 무작위로 깨지고, 파편화된 픽셀 조각이 공기 중으로 떨어져 사라진다. (파괴적 시각화)
    3.  **[Loss Visualization Anchor]:** 화면의 최상단 혹은 중앙 하단에 $X,XXX 금액이 위치한 패널을 강제 노출시킨다. 이 숫자는 **빨간색으로 깜빡이며(High-frequency Flicker)**, 글리치 효과가 발생할 때마다 **'데이터 손실 오류: -XX원'**과 같은 경고 메시지가 함께 플래시 되어야 한다.
    4.  *오디오 힌트:* 높은 주파수의 날카로운 '지지직!' 하는 스파크 사운드와 급격한 볼륨 하락을 반복한다.

#### C. [STAGE 3] Opportunity Gain (The Solution - 골드 단계)
*   **발동 조건:** 사용자가 시스템이 제시하는 해결책(추가 연결/도구 사용 등)을 통해 Missing Edge를 발견하고, 이를 채움으로써 기회비용 손실을 막았을 때.
*   **비주얼 목표:** 공포에서 벗어나 '명확한 가치'와 '안정성'을 되찾는 카타르시스를 제공한다.
*   **애니메이션 스펙 (Lottie/SVG):**
    1.  **[Snap Transition]:** 레드/글리치 상태에서 갑자기 화면이 **'클린하게 재부팅(Clean Reboot)'**되는 듯한 효과와 함께 색상이 앰버 $\rightarrow$ 골드로 급격히 변환된다. (화면 정지 후, 명확한 빛의 확산)
    2.  **[Restored Edge]:** 깨졌던 가장자리들이 마치 자성을 띠는 것처럼 **'강력하게 끌어당겨져(Snap)'** 다시 연결되며, 이 과정에서 골드빛으로 미세하게 번쩍인다. (연결 행위에 대한 보상 시각화)
    3.  **[PVU Confirmation]:** 손실액 패널이 사라지고, 대신 **'추가 순이익 예상: +Y$YY'**와 같은 긍정적인 수치가 황금빛으로 안정적으로 표시된다. 이 숫자는 마치 계좌 잔고처럼 신뢰감을 주어야 한다.
    4.  *오디오 힌트:* 날카로운 스파크 사운드가 부드러운 '딩-' 하는 성공음과 함께 사라지며, 낮은 톤의 안도감 주는 배경 음악으로 전환된다.

### 🖥️ 3. 기술 구현 가이드라인 (Developer Notes)

1.  **컴포넌트 구조:** 모든 애니메이션은 하나의 **`LossFunnelContainer.tsx`** 컴포넌트 내에서 상태(`status: 'PAIN' | 'CRISIS' | 'GAIN'`)에 따라 조건부 렌더링 및 애니메이션 로직을 처리해야 합니다.
2.  **데이터 바인딩:** $X,XXX와 $Y$YY 값은 백엔드 API (코다리 구현 중)를 통해 받아온 **원자적(Atomic) 상태 데이터**로 직접 바인딩되어야 하며, 애니메이션 프레임마다 깜빡임을 동기화해야 합니다.
3.  **성능 최적화:** Lottie/SVG는 반드시 `will-change: transform;` 등의 CSS 속성을 사용하여 부드러운 애니메이션을 보장하고, 특히 글리치 효과는 GPU 가속이 필수입니다.
