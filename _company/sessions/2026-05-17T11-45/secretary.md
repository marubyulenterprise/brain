### 📝 Master Spec v1.0 (내용 구성)

#### I. 개요 및 목표 정의
*   **핵심 가치:** 데이터 통합이 아닌, **정량화된 재무적 손실 경고(Loss Amount)** 경험 제공.
*   **사용자 여정 목표:** 사용자가 자신의 'Missing Edge'를 발견하고 → 그로 인한 충격적인 재무/시간적 손실을 자각하며 → 우리 서비스만이 해결책임을 인식하도록 유도.

#### II. 상태 기계 (State Machine) 정의 (가장 중요!)
이 컴포넌트는 최소 3개의 주요 상태(State)와 명확한 전환 로직(Transition)을 가져야 합니다.

| State ID | 이름 | 설명 및 핵심 목적 | 필수 액션/UI 요소 | 트리거 (Trigger) |
| :---: | :--- | :--- | :--- | :--- |
| **S0** | **초기 진입 / 탐색 (Discovery)** | 사용자가 서비스에 처음 접속하여 데이터를 입력하거나 살펴보기 시작하는 상태. *아직 위기를 인지하지 못함.* | 일반적인 대시보드 UI, 데이터 그래프 시각화. | 스크롤 위치: 10% $\to$ S1 진입 |
| **S1** | **위기 충격 자각 (Shock/Alert)** | 시스템이 'Missing Edge'를 감지하고, 그로 인한 손실액을 극대화하여 경고하는 상태. **(가장 시급함!)** | 🚨 빨간색 경고 오버레이, 카운터 애니메이션으로 $Loss Amount$ 폭발적 제시, 경고 메시지 헤드라인. | Missing Edge 감지 (백엔드 API 호출) |
| **S2** | **해결책 및 통합 제시 (Action/Relief)** | 손실에 대한 충격을 완화하고, 우리 서비스가 이 문제를 어떻게 해결하는지 명확하게 보여주는 단계. | '통합 인덱싱' 프로세스 시각화(노드 재배열 애니메이션), 핵심 기능 튜토리얼 CTA. | 사용자가 경고 메시지를 클릭하거나 스크롤 위치: 70% 이상 도달 |

#### III. 상세 플로우차트 및 컴포넌트 로직 정의
1.  **[S0 $\to$ S1] Transition Logic (위기 유발):**
    *   **조건:** 사용자가 A 데이터셋과 B 데이터셋 간의 논리적 연결 고리가 부족함(Missing Edge)을 감지했을 때.
    *   **액션:** 페이지 로딩 시, 또는 스크롤이 일정 지점에 도달하면 (예: 10% $\sim$ 25%), 부드럽게 S1 상태로 전환하며 **'손실 금액 $X$'**를 애니메이션으로 제시합니다.
2.  **[S1 유지] Shock Component 상세:**
    *   헤드라인: "당신이 놓치고 있는 잠재적 재무 손실액, ₩XXX,XXX원" (가장 크게 강조)
    *   서브 카피: "단순한 데이터 부족이 아닙니다. 비즈니스 로직의 공백입니다."
3.  **[S1 $\to$ S2] Transition Logic (해결책 제시):**
    *   **조건:** 사용자가 경고 메시지(Shock)를 인식하고, 해결 의지를 보였을 때 (클릭 또는 스크롤 하강).
    *   **액션:** 빨간색 위기 분위기가 점차 노란색/파란색 계열의 '질서 정연한' 색상으로 바뀌며, Missing Edge가 우리 서비스의 **[Integration Layer]**를 통해 채워지는 애니메이션을 보여줍니다.

## 🎯 프로젝트 목표 및 핵심 가치
**목표:** 사용자가 자신의 데이터 파편화(Missing Edge)로 인해 발생하는 '재무적 손실액(Loss Amount)'을 극적으로 자각하고, 우리 서비스가 유일한 해결책임을 경험하게 한다.
**핵심 메시지:** "데이터 통합이 아닌, 재무적/시간적 손실 경고 경험에 집중하라."

## ⚙️ I. 상태 기계 (State Machine Diagram) 정의
| State ID | 이름 | 설명 및 목적 | 트리거 조건 | 주요 UI 컴포넌트 & 액션 |
| :---: | :--- | :--- | :--- | :--- |
| **S0** | **[Discovery] 탐색 단계** | 사용자가 데이터를 관찰하며 문제의 존재를 인지하지 못하는 상태. (Baseline) | 초기 진입 및 스크롤 시작 | 일반 대시보드, 데이터 시각화 그래프, 낮은 긴장도의 정보 제공. |
| **S1** | **[Shock] 위기 자각 단계** | 시스템이 Missing Edge 감지를 통해 손실액을 폭발적으로 제시하며 경고하는 상태. **(가장 중요)** | 1. 백엔드에서 'Missing Edge' API 호출 성공. 2. 사용자 스크롤 위치: 10% $\to$ 30%. | **[🚨 Alert Component]** 빨간색 오버레이, $Loss Amount$ 카운터 애니메이션 (가장 큰 폰트), 충격적인 헤드라인. |
| **S2** | **[Action/Relief] 해결책 제시 단계** | 위기 자각 이후, 서비스의 통합 기능을 통해 손실을 회복할 수 있다는 안도감을 제공하는 상태. | 1. 사용자가 [Alert Component] 내 CTA 버튼 클릭. 2. 스크롤 위치: 60% $\to$ 90%. | **[✨ Integration Flow]** 노드 재배열 애니메이션, 단계별 API 호출 플로우 차트 제시, 해결책 기반의 명확한 Next Step 제시. |

## 🗺️ II. 사용자 여정 (User Journey) 및 플로우차트 상세
### 1. S0 $\to$ S1 Transition: 위기 유발 (The Shock Moment)
*   **Trigger:** Missing Edge 감지 로직 발동 (API 호출 성공).
*   **Action Flow:** 스크롤 위치가 임계점(Threshold)을 넘어서면, 부드러운 시각적 전환과 함께 S1 상태의 경고 컴포넌트가 화면에 오버레이 되어야 합니다.
*   **Visual Guideline:** 배경은 어두워지거나 채도가 낮아지고, 빨간색 계열의 대비 색상을 사용합니다. (경계심 유발)

### 2. S1 유지: 충격 강조 컴포넌트 상세 스펙
*   **헤드라인 카피:** "🚨 [Missing Edge] 발견! 당신이 지금 놓치고 있는 잠재적 재무 손실액은 **₩[Loss Amount]** 입니다."
    *   *(강조 포인트: Loss Amount를 최대 폰트로, 애니메이션 효과와 함께 제시)*
*   **본문 카피:** "단순한 데이터 공백이 아닙니다. 이 연결 고리의 부재는 곧 비즈니스 로직의 치명적인 손실로 이어집니다."
*   **CTA (Call To Action):** [💡 해결책 확인하기] 버튼. (S1 $\to$ S2 전환을 유도)

### 3. S2: 해결책 제시 및 회복 경험 (The Relief & Solution)
*   **Transition Trigger:** 사용자가 CTA를 클릭하거나, 충분히 충격을 받은 후 다음 단계를 탐색함.
*   **핵심 애니메이션:** 파편화되어 보이던 노드들이 **[Maru Integration Layer]**를 거쳐 질서정연하게 재배열되는 모습을 시각적으로 보여줍니다 (통합의 느낌).
*   **메시지:** "Missing Edge? 걱정 마세요. Maru Company가 전문 엔진을 통해 이 공백을 완벽히 메우고, 잠재력을 회복시켜 드립니다."

## 🔑 III. 개발팀 가이드라인 및 기술적 고려 사항
