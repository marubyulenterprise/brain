## 📈 P0 핵심 기능 크레딧 소모 로직 (Credit Consumption Logic)

크레딧은 사용자가 입력한 **'원천 자산(Source Asset)'**의 가치와, 시스템이 수행하는 **'처리 난이도/출력 구조화 수준'**에 비례하여 책정되었습니다.

| 기능 분류 | 세부 단위 (Output Scope) | 측정 기준 및 상세 설명 | 크레딧 비용 (Cost per Unit) |
| :--- | :--- | :--- | :--- |
| **1. 콘텐츠 재가공 (Content Repurposing)** | **블로그 초안 생성** | 1,000자~2,000자 분량의 구조화된 개요 및 본문 초안 1회. (예: 스크립트 → 블로그 포맷) | **5 크레딧** |
| | **소셜 미디어 콘텐츠 패키지** | 하나의 원천 자산에서 추출 가능한 트위터/인스타그램용 짧은 문구 3~5개 세트. | **2 크레딧 (세트당)** |
| | **비디오 스크립트 아웃라인 생성** | 특정 주제에 대한 영상 기획의 논리적 흐름(도입-전개-결론)을 포함한 구조화된 목차 초안 1건. | **7 크레딧** |
| **2. 지식 베이스 추출 (KB Extraction)** | **핵심 키워드 요약 및 정리** | 오디오/영상 전체 내용 중 핵심 주제어와 관련 문구 목록 추출. 가장 기본적인 정보 처리 단위. | **1 크레딧** |
| | **Q&A 페어 정의** | 원천 자산의 내용을 기반으로 질문(Q)과 답변(A) 쌍 3~5개 구조화 요청 1건. (가장 많이 사용될 예상 기능). | **3 크레딧** |
| | **체계적 지식 지도 구축 (KB Map)** | 방대한 양의 원천 자산을 분석하여 계층적/분류학적 관계를 가진 최종 지식 베이스 구조(Markdown 형태) 1건. | **8 크레딧** |
