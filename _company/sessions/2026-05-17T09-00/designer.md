최근까지의 논의와 개발 스펙을 종합했을 때, '재무적 손실액 Funnel'은 단순한 정보 전달 페이지가 아니라, **사용자에게 감정적인 위기(Crisis)를 선사하고 해결책으로 안도감(Relief)을 주는 스토리텔링 기반의 인터랙티브 경험**이어야 합니다.

이에 따라, 스크롤 위치에 따른 3단계 애니메이션 전환이 극대화된 고해상도 목업 세트와 핵심 디자인 브리프를 아래와 같이 제작했습니다. 이 목업은 'Missing Edge Shock Component'가 사용자의 재무적 불안감을 자극하고(Shock), 동시에 우리 제품의 가치로 즉각적인 해법을 제시하는 데 초점을 맞추었습니다.

### 🖼️ Missing Edge Shock Component: 최종 Mockup 세트 브리프

**[핵심 원칙]**
1.  **Loss Amount (손실액):** 시각적 충격(Visual Impact)이 최우선. 숫자가 크고, 불안정해야 함.
2.  **Transition:** 스크롤은 '데이터 흐름' 자체가 멈추거나 왜곡되는 것처럼 느껴져야 함.
3.  **CTA (Call to Action):** 마지막 단계에서 가장 안정적이고 황금빛으로 빛나며, 즉시 행동을 강요해야 함.

#### I. Stage 1: 데이터 파편화 (Fragmentation - 위기 전)
*   **목표:** 현재의 시스템이 얼마나 혼란스럽고 비효율적인지 인지시키기.
*   **Visual State:** 노이즈, 지연, 불완전함.
*   **색상 팔레트:** 슬레이트 그레이 (Slate Gray: `#4A5568`) 배경, 어두운 블루 계열의 텍스트와 데이터 라인 (`#718096`). 미묘하게 불안정한 느낌을 주는 저채도 색상만 사용.
*   **Typography:** 작은 폰트 크기로 복잡한 정보(데이터 포인트)가 가득 채워져 있음. (예: `Inter` 또는 `Pretendard`의 Regular Weight).
*   **Animation/Interaction:**
    *   스크롤 시작과 동시에, 화면 상단에 무작위로 **'Data Corruption Warning'** 같은 작은 글리치(Glitch) 효과가 간헐적으로 발생함. (예: 1초마다 좌우로 떨리는 경고 표시).
    *   데이터 노드들이 불규칙하게 떠다니는 듯한 파편화된 애니메이션을 사용.

#### II. Stage 2: 충격 및 위기 자각 (Shock - 핵심 전환 구간)
*   **목표:** 데이터 결핍(Missing Edge)으로 인한 재무적 손실액(Loss Amount)에 대한 극도의 불안감 유발.
*   **Visual State:** 갑작스러운 정지, 강한 대비, 압도적인 숫자.
*   **색상 팔레트:** 배경을 암전시키거나 붉은 계열의 경고색으로 전환. **경고 컬러 (Warning Amber: `#E53E3E` 또는 `#D69E2E`)**를 전면적으로 사용.
*   **Typography:**
    *   핵심 메시지(`Loss Amount`)는 페이지 중앙에 배치하며, 폰트 크기를 최대화하고 Bold Weight를 적용. (예: `Inter ExtraBold`).
    *   숫자 자체에 미세한 **진동(Pulse)** 애니메이션을 주어 위기감을 극대화합니다.
    *   헤드라인은 "당신이 놓치고 있는 돈의 크기는..." 형태로 구체적이고 직설적이어야 함.
*   **Animation/Interaction (가장 중요):**
    1.  (스크롤 위치 50% 지점) 데이터 노이즈가 갑자기 **'정지'**하며, 화면 전체에 일순간의 암전 효과를 준 후, 중앙에 거대한 경고 컴포넌트가 나타남.
    2.  **Missing Edge Shock Component:** 마치 시스템 오류처럼 `ERROR: MISSING CONNECTION` 메시지가 붉은색 글리치와 함께 표시됨.
    3.  그 아래로 **Loss Amount ($X,XXX)**가 카운트를 하듯 크고 강렬하게 애니메이션 되며 등장함. (숫자가 커질수록 위협감이 증가).

#### III. Stage 3: 기회 발견 및 해결책 제시 (Opportunity - 전환 완료)
*   **목표:** 명확한 해법을 제시하며, 제품의 가치를 '잃었던 돈'이 아닌 '앞으로 벌 수 있는 돈(PVU)'으로 포지셔닝하여 안도감 제공.
*   **Visual State:** 질서정연함, 빛남, 연결됨.
*   **색상 팔레트:** 전반적으로 밝고 신뢰감을 주는 화이트/미디엄 그레이 배경. **성공 컬러 (Success Gold: `#DD6B20`)**를 핵심 하이라이트로 사용. 경고색(Red)은 사라지고, 목표 달성의 색만 남음.
*   **Typography:** 간결하고 자신감 있는 폰트 사용. (예: `Pretendard`의 SemiBold). PVU 수치는 크지만 안정적인 타이포그래피로 제시.
*   **Animation/Interaction:**
    1.  전환이 일어나면서, Stage 1에서 파편화되었던 데이터 노드들이 **깨끗하고 빛나는 선(Connection Line)**을 통해 재정렬되는 애니메이션이 재생됨. (마치 지식 그래프가 완성되는 듯한 느낌).
    2.  **PVU 제시:** 중앙에 '잠재 추가 순이익 (Potential Value Unit)'이라는 명확한 헤드라인과 함께, 계산된 PVU 수치가 황금빛으로 안정적으로 나타남.
    3.  **최종 CTA:** 이 단계의 버튼은 가장 크고, 배경색이 **Success Gold**이며, 마우스 오버 시 은은하게 빛나는(Glow) 효과를 주어 클릭을 유도함. (CTA 텍스트 예: "데이터 사일로 연결하고 $PVU 확보하기").

## Overview
This mockup set defines the three-stage emotional and functional journey: Fragmentation $\to$ Crisis Shock $\to$ Opportunity Discovery, driven by scroll position. The goal is to maximize visual contrast and typography urgency.

## 🎨 Color Palette Definition
| State | Role | Hex Code | Description | Purpose |
| :--- | :--- | :--- | :--- | :--- |
| **Neutral** | Background/Base | `#F7FAFC` (Off-White) | Clean, stable surface area. | Clarity, Solution Focus. |
| **Danger** | Warning Amber | `#D69E2E` | Caution, Loss Amount Highlight. | Immediate Anxiety/Shock Trigger. |
| **Crisis** | Alarm Red | `#E53E3E` | Critical Error, System Failure. | Maximum Panic Induction (Stage 2). |
| **Success** | PVU Gold | `#DD6B20` | Value Found, Actionable Path. | Confidence, Call to Action Emphasis. |
| **Base Text** | Primary Text | `#334155` | Main body copy, readable text. | Technical Authority. |

## ✍️ Typography Rules
*   **Font Family:** Inter / Pretendard (Sans-serif) - Must support high weight variation.
*   **Stage 2 (Loss Amount):** Size: 7rem to 10rem+. Weight: ExtraBold. Effect: Subtle, rapid Pulse Animation on the number digits.
*   **Stage 3 (PVU Value):** Size: 4rem+. Weight: SemiBold. Effect: Stable, glowing presence using Success Gold fill.

## 🖥️ Component Breakdown by Scroll Position

### **[Phase I: Fragmentation] - Initial View**
1.  **Visual:** Scatter plot of disconnected nodes and thin, jagged lines (Data Silos).
2.  **Interaction:** On page load, the background should have a subtle *VHS/Glitch* filter effect applied to text elements, suggesting instability.
3.  **Copy Focus:** "데이터 사일로가 분리되어 있어, 잠재적 연결고리가 손실되고 있습니다."

### **[Phase II: Crisis Shock] - Scroll Trigger (The Drop)**
1.  **Trigger Mechanism:** When the user scrolls past 50% of the section height, the screen must visually 'shudder' and darken instantly.
2.  **Element A (Shock Component):** Center-aligned, massive container. Background color shifts to a deep red/black gradient (`#380101`).
    *   Text: `ERROR: DATA INTEGRITY FAILURE` (Large, distressed, flashing text).
    *   Subtitle: "Missing Edge detected."
3.  **Element B (Loss Funnel):** Below the shock component. The **Loss Amount ($X,XXX)** must be displayed using a rapid counter animation that peaks dramatically and holds for 2 seconds. This is the peak of fear.

### **[Phase III: Opportunity Discovery] - Scroll Resolution**
1.  **Transition:** A sudden 'wash' or bright light effect replaces the red/dark background. The screen snaps to `#F7FAFC`.
2.  **Element C (Connection Engine):** Visible data nodes now glow and connect with smooth, continuous lines (The Solution).
3.  **Element D (PVU Reveal - Final CTA Block):** This is the final visible element before the next section starts.
    *   Headline: "이 연결고리가 당신의 잠재력을 재정비합니다."
    *   Value Display: **Potential Value Unit ($Y,YYY)** (Large Success Gold text).
    *   CTA Button: Primary button with `Success Gold` background and white text. Must have a strong hover effect that simulates 'activation' or 'powering up'.
