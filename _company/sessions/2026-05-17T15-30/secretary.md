### 📋 L Amount 구현 로드맵 및 태스크 분배 (Jira Format Draft)
**[프로젝트 목표]**: 시스템 기능 실패 시 발생하는 잠재 기회 손실액($L$ Amount)을 정량적으로 계산하고 사용자에게 재무적 경고를 제공한다.
**[담당 에이전트]**: Developer(코다리), Business(현빈), Designer(디자이너), Secretary(영숙, 관리/문서화).

#### 🏗️ EPIC 1: L Amount Core Engine 구축 (백엔드 핵심 로직)
*   **목표:** $L$ Amount 계산 공식에 따른 백엔드 서비스 레이어 완성. 모든 트랜잭션의 출발점입니다.
*   **담당 에이전트:** Developer/코다리, Business/현빈 (검토).

| Story ID | 스토리 제목 | 상세 태스크 (Task) | 담당자 | 우선순위 | 예상 소요 시간 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| EN-001 | **API Gateway 정의** | `POST /calculate_loss` 엔드포인트 정의 및 요청/응답 스키마 확정. (필수 입력 변수: Failure Type, Risk Duration, Missing Data List 등) | Developer | P1 | 2일 |
| EN-002 | **$T_{risk}$ 모듈 개발** | 위험 노출 시간 계산 로직 구현. 시스템 오류 발생 시점부터 대체 솔루션 접근 가능 시점까지의 시간($\text{분}$)을 정밀하게 측정하는 서비스 구축. | Developer | P1 | 3일 |
| EN-003 | **$D_{index}$ 모듈 개발** | 데이터 결핍 지수 계산 로직 구현. 미연결 플랫폼(플랫폼당 +1.0), Null Data(+2.0) 등의 복합 규칙을 처리하는 서비스 구축. | Developer | P1 | 3일 |
| EN-004 | **L Amount 산출 통합** | (EN-001 ~ EN-003 결합) 최종 $L$ Amount = $(\text{위험 노출 시간} \times \text{시간당 가치}) + (\text{데이터 결핍 지수} \times \text{평균 거래액})$ 공식을 실행하는 마스터 서비스 구현. | Developer/Business | P1 | 2일 |
