CEO 지시사항을 검토했습니다. 단순히 API 게이트웨이의 데이터를 보여주는 것이 아니라, 그 데이터가 사용자에게 **'위기감(Urgency)'**과 **'필요한 행동(Actionability)'**으로 전환되는 전체 UX 흐름을 설계하는 것이 핵심입니다.

아래는 **[1인 기업 운영 대시보드]** 와이어프레임 및 UX/UI 디자인 브리프입니다. 이 구조는 개발팀이 즉시 컴포넌트화하여 구현할 수 있도록 색상 코드, 상태 로직, 애니메이션까지 명세했습니다.

## 💡 프로젝트명: Maru Dashboard - [Critical Loss Monitor]
**목표:** 사용자가 시스템에 접속하는 순간, 현재의 '미활용된 자산'이나 '구조적 결함'으로 인한 재무적 손실액($L$ Amount)을 가장 먼저 인지하게 한다.

### 🎨 디자인 시스템 브리프 (Color & Typography)

| 요소 | 상태/상황 | 컬러 코드 (HEX) | 역할 및 의미 |
| :--- | :--- | :--- | :--- |
| **Primary** | 기본 액션, 성공 | `#2E7D32` (Dark Green) | 회사 신뢰도, 긍정적 발전, 핵심 CTA |
| **Warning/PVU** | 잠재 기회 발견 | `#FFC107` (Amber/Gold) | 경고가 아닌 '기회' 제시. 적극적인 탐색 유도. |
| **Critical Red** | **손실 임계치 초과** | `#DC2E2E` | **즉각적 위협, 손실 발생.** 가장 높은 시각적 대비와 충격 필요. |
| Background | 기본 배경 | `#F9FAFB` (Off-White) | 눈의 피로를 줄이고 컴포넌트 분리 효과 극대화. |
| Typography | 주 폰트 | Inter, Pretendard (산세리프 권장) | 기술적, 모던한 느낌 유지. |

### 🖥️ 와이어프레임 구조 및 레이아웃 명세 (Dashboard Layout)

**A. 페이지 구성:** SaaS Dashboard 표준 레이아웃을 따릅니다.
*   `[1]` 사이드바: 네비게이션 영역 (항상 고정, Primary 컬러 사용).
*   `[2]` 탑바: 사용자 정보, 알림(🔔), 데이터 동기화 상태 표시.
*   `[3]` 메인 컨텐츠 영역: 위기감 조성 플로우가 발생하는 핵심 공간.

**B. 컴포넌트 상세 설계 및 흐름 (The Crisis Flow)**

#### 1️⃣ [최상단 - Hero/Alert Zone]: **Critical Loss Monitor Card (필수)**
*   **위치:** 메인 컨텐츠 영역 최상단, 가장 넓은 가로 폭을 차지합니다. (시각적 압도감 최대화)
*   **트리거:** API Gateway가 $L$ Amount를 계산하여 '임계치 초과' 상태(Critical Red)를 보고할 때만 이 컴포넌트가 활성화됩니다.
*   **디자인 요소:**
    *   배경: `#DC2E2E` (Critical Red). 배경 전체에 미세한 `Pulse & Flicker` 애니메이션을 적용하여 경고의 지속성을 강조합니다.
    *   헤드라인 (`H1`): "🚨 데이터 결함으로 인한 예상 손실 감지" (텍스트는 흰색 `#FFFFFF`).
    *   핵심 지표: **"$L$ Amount:** $₩12,500,000$" (글자 크기 최대화, 굵게). 이 금액은 애니메이션 효과(카운트업)와 함께 나타나야 합니다.
    *   보조 설명: "이 손실액은 현재 [A] 기능의 데이터 입력 누락 및 구조적 패턴 분석 미흡에서 기인합니다." (기술적/정밀한 톤 유지).

#### 2️⃣ [중간 섹션]: **Loss Breakdown & Root Cause Analysis**
*   **목표:** 위기감을 학문적으로 해체하여 신뢰도를 높입니다.
*   **레이아웃:** 3열 그리드(Grid)를 사용합니다.
*   **컴포넌트 (A, B, C):**
    *   **[A] Missing Edge 지점:** 어떤 데이터 연결이 끊어졌는지 시각화합니다. (노드 그래프 형태로 보여주되, 손실된 엣지 부분만 `Critical Red`로 처리하고, 해당 위치에 '??? - 결함' 아이콘을 배치).
    *   **[B] 시간/자원 누수:** 자원이 비효율적으로 사용되는 지점을 표시합니다. (예: "자동화 미적용 리스크 구간", 시각적으로 막힌 파이프라인 이미지와 함께 손실률(%) 제시).
    *   **[C] 최적 기회 상실:** 이 데이터를 활용했다면 얻을 수 있었던 잠재 수익(PVU)과 현재 손실액($L$ Amount)을 비교하여 보여줍니다. (그래프 형태: Potential Peak $\rightarrow$ Actual Low Line / Gap = $L$ Amount).

#### 3️⃣ [최하단 - Action Zone]: **The Solution Funnel & CTA**
*   **목표:** 위기감을 느낀 사용자를 즉시 다음 행동으로 유도합니다. (가장 중요한 전환 지점)
*   **핵심 컴포넌트: '즉각적 리스크 방어 및 최적화 패키지' 카드.**
    *   배경색: `#F9FAFB`에서 약간 더 어둡거나, 신뢰감을 주는 톤으로 변경합니다.
    *   제목: "✅ 이 손실을 막고 성장을 지속하기 위한 다음 단계" (해결책 제시).
    *   **CTA 버튼:** `[Pro Tier 업그레이드 및 데이터 정밀 분석 시작]`
        *   스타일: Primary Color (`#2E7D32`) 사용.
        *   효과: 이 버튼은 위기 경고가 사라지면(즉, 가입이 이루어지거나 대안을 선택하면) 가장 눈에 띄게 활성화되어야 합니다.
