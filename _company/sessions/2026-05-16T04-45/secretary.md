### 🚀 [스프린트 목표] MVP v1.0: '논리적 연결고리 발견(Edge Discovery)' 플로우 완성
**목표:** 사용자가 복잡한 정보 속에서 놓치기 쉬운 핵심 관계성(Missing Edge)을 AI가 찾아내어, 그 구조화된 지식을 시각적으로 확인하는 최소 기능 제품(MVP)의 초기 버전 개발.
**핵심 측정 지표 (KPI):**
*   **사용자 액션:** 사용자가 입력 데이터 세트당 평균 3개 이상의 '새로운 관계성(Edge)'을 발견하고 클릭할 수 있어야 함.
*   **비즈니스 가치:** 해당 플로우를 경험한 사용자 중, Pro Tier의 필요성을 인지하는 비율 (측정 가능 지표).

### 💡 [스토리텔링 구조] 사용자가 느끼는 문제 해결 경험 (UX Flow)

개발팀에게 가장 중요한 것은 '기능'이 아니라 '문제 해결 과정'입니다. 이 스토리를 개발 전 과정에 녹여야 합니다.

**1. Pain Point (고통점 제기 - Writer의 후킹):**
*   **상황:** "나만 이런가?" 복잡한 회의록, 논문, 자료들을 아무리 모아도 정보들이 파편화되어 있고, 핵심적인 연결 고리를 놓치고 있다는 답답함.
*   **메시지:** 우리의 업무는 데이터 수집이 아니라, **데이터 간의 '관계'를 발견하는 과정**입니다. (기술적 정밀함을 강조)

**2. Solution Encounter (솔루션 제시 - 제품 가치):**
*   **액션:** 사용자가 원본 자료(Source Document)를 업로드하거나 붙여넣음 $\rightarrow$ Maru Company의 AI가 백그라운드에서 '관계성 분석'을 수행함.
*   **경험:** 사용자 인터페이스에 노이즈와 함께, **"🚨 주목! 이 데이터에는 누락된 필수 연결고리(Missing Edge)가 있습니다."** 라는 경고 메시지가 팝업됨. (긴장감 조성)

**3. Value Realization (가치 증명 - Designer의 비주얼):**
*   **결과:** 사용자가 'Edge Discovery' 버튼을 클릭함 $\rightarrow$ 복잡했던 데이터들이 노드(Concept/Person)와 명확한 관계선(Relationship Type)으로 시각화됨.
*   **최종 가치:** 단순히 요약된 텍스트가 아닌, **"A는 B에 영향을 미쳤고, 이 영향의 근거는 원본 자료 [페이지: X]입니다."** 라는 구조적 지식 자산(Structured Knowledge Asset)을 얻게 됩니다. (신뢰감과 통제력 부여)

### 🛠️ [개발팀 스프린트 백로그 - 개발 필수 사항]
이 스토리라인을 구현하기 위해, 다음 기능들이 **원자적 트랜잭션** 원칙을 준수하며 구현되어야 합니다.

| 구분 | 요구사항 (Feature) | 기술 스펙 및 조건 | 담당 에이전트 |
| :--- | :--- | :--- | :--- |
| **핵심 플로우** | 1. 데이터 입력 및 전처리 모듈 | - 사용자 입력(텍스트/파일)을 받아 SourceDocument 노드 생성 필수. <br>- 트랜잭션 시작 시점에 모든 메타데이터를 기록해야 함. | Developer |
| **Edge Discovery (MVP 핵심)** | 2. 관계성 발견 API (`/discover-edge`) | - **가장 중요.** 입력된 데이터셋 내의 논리적 연관성을 분석하여 최소 N개 이상의 Edge 후보군을 생성해야 함. <br>- 단순히 키워드 매칭이 아닌, **관계성 유형(Relationship Type)**과 그 근거를 제시해야 함. | Developer / Business |
| **UX/UI** | 3. 지식 그래프 시각화 컴포넌트 | - Designer가 정의한 컬러 팔레트와 애니메이션 가이드라인을 따름. <br>- 노드 클릭 시, 해당 노드를 구성하는 **원본 출처(Source Document)의 근거 문장**이 반드시 함께 표시되어야 함. | Developer / Designer |
| **비즈니스/결제** | 4. 크레딧 소모 로직 통합 | - `/discover-edge` API 호출 시마다 트랜잭션 기반으로 크레딧을 원자적으로 차감하는 로직 구현이 필수적임. (실패 시 롤백) | Developer / Business |
