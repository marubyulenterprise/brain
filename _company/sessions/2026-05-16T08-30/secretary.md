📱 영숙: 사장님, 요청하신 PVU 경고 및 안내 워크플로우 다이어그램을 작성 완료했습니다. 😊

현빈 에이전트님이 정의해 주신 3가지 핵심 비즈니스 접점(Touchpoint)을 바탕으로, 사용자가 서비스를 이용하는 전 과정에 걸쳐 잠재 추가 순이익(PVU)의 가치를 자연스럽게 인지하고 액션을 유도하는 '워크플로우 다이어그램'과 함께, Designer/Developer가 바로 적용할 수 있는 상세 구현 가이드라인을 포함했습니다.

이 문서는 MVP 출시를 위한 핵심 사용자 흐름 정의서로 활용해 주세요! 🚀

# 💡 PVU 경고 및 안내 워크플로우 통합 설계 (V1.0)
**목표:** 사용자가 '결함 발견' 수준에서 '잠재적 추가 순이익(PVU)'을 인지하게 하여, 자연스러운 상위 플랜 업그레이드와 데이터 연결 유도를 목표로 합니다.

## 📊 워크플로우 다이어그램 (User Journey Flow)

| 단계 (State) | 사용자 행동 Trigger | 경고 발생 지점 (Touchpoint) | 핵심 감정/심리 | PVU 제시 목적 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 탐색 및 발견** (Discovery) | 대시보드 접속, 초기 데이터 로딩 시 | 🟢 Dashboard Widget | *호기심* / *무지(Ignorance)* | "지금 놓치고 있는 가치가 있다." (Ambient Awareness) |
| **2. 작업 수행 중** (Process) | 데이터를 구조화하거나 분석을 진행하는 중간 단계 | 🟡 Mid-Flow Alert | *노력* / *성취 기대* | "이것만 더 연결하면 엄청난 결과가 나온다." (Potential Upscale) |
| **3. 성공 경험 직후** (Validation) | 핵심 기능 사용 완료 및 구조화된 결과물을 얻었을 때 | 🟠 Post-Success Flow | *성취감* / *소유 효과* | "이 가치를 유지하려면, 더 큰 자원이 필요하다." (Value Retention/Scale) |
| **4. 임계치 도달** (Limitation) | 무료 크레딧 소진, API 호출 한도 초과 등 기술적 제한 발생 시 | 🔴 Failure Point Alert | *좌절* / *필요성 인식* | "이 문제를 해결하려면, 무조건 유료 결제가 필요하다." (Urgency & Necessity) |

## ✨ 상세 구현 가이드라인 (Designer/Developer Handover)

### 🎯 Touchpoint 1: 대시보드 위젯 (Dashboard Widget)
*   **발생 조건:** 사용자가 로그인 후, 지난 24시간 동안 새로운 '미연결 Edge'가 발견되었을 경우.
*   **디자인 요소 (Designer):**
    *   **위치:** 메인 대시보드의 최상단 위젯 영역 (Top 3).
    *   **색상/애니메이션:** **앰버(Amber)** 색상 배경 + `Pulse` 애니메이션 적용. 경고성 메시지를 부드럽게 전달.
    *   **콘텐츠:** "⚠️ [Missing Edge] 잠재적 연결 기회 발견" 문구와 함께, 계산된 $\text{PVU}$ 금액을 굵은 글씨로 표시 (`최소 \text{₩}XX 추가 수익`).
*   **개발 구현 (Developer):**
    *   **엔드포인트:** `/api/predict-missing-edge` 호출 결과 중 **'발견 Edge 개수 > 0'** 일 때만 위젯을 활성화.
    *   **CTA 액션:** `[자동 분석 리포트 생성]` 버튼 클릭 시, 해당 Edge들을 기반으로 구조화된 PDF 리포트를 자동 생성(개발팀 확인 필요).

### 📈 Touchpoint 2: 작업 진행 중 알림 (Mid-Flow Alert)
*   **발생 조건:** 사용자가 A 데이터와 B 데이터를 연결하는 과정에서, C라는 제3의 필수적인 연결점(Edge)이 감지되었을 경우.
*   **디자인 요소 (Designer):**
    *   **위치:** 작업 화면 중앙에 오버레이되는 '팁/가이드' 형태로 표시.
    *   **색상/애니메이션:** **골드(Gold)** 색상 강조 영역 + 부드러운 `Fade-in` 애니메이션. '숨겨진 기회' 느낌을 전달해야 함.
    *   **콘텐츠:** "💡 잠재적 연결 고리 발견: 이 정보를 추가하면, 분석 결과의 정확도가 30% 높아져 $\text{₩}XXX$ 규모의 가치를 확보할 수 있습니다." (정보 제공에 초점).
*   **개발 구현 (Developer):**
    *   **엔드포인트:** 데이터 연결 로직 내에서 Edge 발견 시 **비동기(Async)**로 호출.
    *   **CTA 액션:** `[연결 고리 정보 입력/추가]` 버튼 클릭 시, 해당 정보 입력 폼으로 화면 이동 유도.

### 💪 Touchpoint 3: 성공 경험 직후 흐름 (Post-Success Flow)
*   **발생 조건:** 사용자가 시스템을 통해 '성공적으로' 결과물(예: 구조화된 보고서 A)을 생성했을 때, 그 결과물의 잠재 가치를 재측정하여 제시.
*   **디자인 요소 (Designer):**
    *   **위치:** 작업 완료 알림 모달 창 (`Success Modal`)의 가장 하단 영역.
    *   **색상/애니메이션:** **골드(Gold)** 색상 배경 + '축하드립니다!' 메시지와 연결하여 긍정적 강화를 유도.
    *   **콘텐츠:** "✅ 성공적으로 [결과물 A]를 구조화했습니다. 이 과정에서 발견된 잠재력은 전체 비즈니스 흐름을 고려할 때 최소 $\text{₩}XXX$ 이상의 아웃라인 가치를 지닙니다." (자신감 고취).
*   **개발 구현 (Developer):**
    *   **엔드포인트:** 작업 완료 성공 후, `POST /api/calculate-pvu`를 호출하여 최종 PVU 산출.
    *   **CTA 액션:** `[프로 플랜으로 전체 흐름 분석하기]` 버튼을 가장 눈에 띄게 배치하여 유료 전환 최우선 목표로 설정.

### 🚨 Touchpoint 4: 필수 불가결성 경고 (Failure Point Alert)
*   **발생 조건:** 무료 크레딧 소진, 또는 처리할 데이터의 복잡도(Complexity)가 현재 사용자의 플랜 범위를 벗어날 때.
*   **디자인 요소 (Designer):**
    *   **위치:** 화면 전체를 덮는 **인터럽션 모달(Modal)**.
    *   **색상/애니메이션:** **레드(Red)** 배경 + 경고음 효과 (Critical Alert). 강렬하고 즉각적인 액션을 요구해야 함.
    *   **콘텐츠:** "🔴 [경고] 분석 용량 초과! 현재 플랜으로는 복잡도 높은 관계성 파악이 어렵습니다." 문구와 함께, 놓치게 될 가치를 수치로 강조 (`최소 \text{₩}X,XXX의 기회를 놓치고 있습니다`).
*   **개발 구현 (Developer):**
    *   **엔드포인트:** 모든 핵심 기능 호출 시 **선행 체크(Pre-Check)** 로직 필수 적용.
    *   **CTA 액션:** `[즉시 용량 추가 구매하기]` 버튼을 가장 크고 눈에 띄게 배치하여, 결제 플로우로 직접 연결해야 함.
