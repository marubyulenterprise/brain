## 🎬 Animation Production Blueprint: PVU 발견 과정 시각화 (3단계)
**목표:** 복잡한 데이터 단절(Missing Edge)이 어떻게 '잠재 추가 순이익(PVU)'으로 계산되고, 최종적으로 재무적 손실액($)으로 변환되는지를 사용자에게 가장 충격적이면서도 신뢰감 있게 전달한다.
**톤앤매너:** 기술적 정밀함 (Technical Precision), 위기감 조성 (Urgency), 해결책 제시의 명확성 (Clarity).
**총 예상 시간:** 60초 ~ 90초 (핵심 PVU 계산 구간에서 속도 조절 가능)

---

### **[STAGE 1] 🧩 Pain Point 제시: 데이터 단편화와 혼란 (The Fragmentation)**
*(시간 배분: 0초 ~ 20초)*

| 요소 | 내용 및 스토리보드 지시 | 시각적/기술적 구현 브리프 | 비즈니스 목표 |
| :--- | :--- | :--- | :--- |
| **오프닝 후크** | (내레이션) "당신의 데이터는 충분한가요? 아닙니다. 파편화되어 있습니다." 화면에 사용자 업무 과정(예: 미팅 녹취록, 기획안 초안, 고객 피드백 등)의 텍스트 조각들이 무질서하게 플로팅 된다. | **비주얼:** 어둡고 복잡한 그리드 배경. 수많은 작은 노드(데이터 포인트)가 카오스처럼 움직임. 색상 팔레트는 회색/청회색 계열 (신뢰도 하락). | 사용자의 즉각적인 문제 인식 유발 (Pain Point). |
| **흐름 시각화** | 데이터 조각들이 특정 '주제'나 '사람'을 중심으로 모여야 할 자리에, 마치 전선이 끊긴 것처럼 공백(Void)이 생긴다. 이 공백에 포커스를 맞춘다. | **애니메이션:** 노드 A $\rightarrow$ (연결 시도) $\rightarrow$ 노드 B가 되어야 하는데, 중간에 투명하고 강력한 '끊김' 애니메이션(`Broken Edge` 효과) 발생. 배경음은 불협화음을 유지한다. | 데이터의 연결성 부족을 직관적으로 인식시킴. |
| **핵심 메시지** | (내레이션) "이 단절된 연결고리, 즉 Missing Edge가 당신이 놓치고 있는 잠재 자산입니다." | 화면 중앙에 'Missing Edge'라는 텍스트와 함께 `?` 아이콘이 크게 등장한다. 주변의 데이터 노드들이 이 공백을 향해 끌어당겨지는 듯한 시각 효과를 적용한다. | 문제 정의 및 서비스 필요성 각인. |

---

### **[STAGE 2] 📉 Loss Quantification: 기회비용 손실액 계산 (The Void)**
*(시간 배분: 21초 ~ 45초)*

| 요소 | 내용 및 스토리보드 지시 | 시각적/기술적 구현 브리프 | 비즈니스 목표 |
| :--- | :--- | :--- | :--- |
| **전환점** | (내레이션) "이 공백은 단순한 정보 누락이 아닙니다. 재무적 기회비용 손실입니다." 화면의 분위기가 갑자기 정지하며, 배경에 '₩' 또는 '$' 심볼이 희미하게 번쩍인다. | **애니메이션:** 카오스하던 노드들이 멈추고, 공백 부분에만 강력한 빨간색/주황색 하이라이트가 들어간다. (경고의 시각화). | 문제의 규모를 감성적/재무적으로 연결. |
| **손실 프로세스** | Missing Edge가 발견된 노드들을 기반으로, 가상적인 '최적의 데이터 플로우'가 역추적된다. 이 과정에서 각 단절 지점마다 예상되는 기회비용 손실액이 계산되어 플로팅 된다. | **UI/UX:** 화면 좌측에 `[Loss Calculator]` 위젯 등장. (Developer Mockup 참조). 노드 A와 C 사이에 'Missing Edge'가 감지될 때, 즉시 `$XXX` 금액이 팝업되며 흐르는 애니메이션 적용. (예: $12,300 / 건당) | 데이터 손실을 구체적인 숫자로 인식시켜 불안감 증폭. |
| **결과 제시** | 모든 손실액을 합산하여 화면 중앙에 거대한 '잠재적 기회비용 손실 총액'이 빨간색으로 깜빡이며 표시된다. | **애니메이션:** `$XX,XXX,XXX` (빨강). 이 수치는 시각적으로 사용자의 눈길을 강탈해야 하며, 배경에서 규칙적인 `Pulse & Flicker` 효과가 적용되어 위기감을 유지한다. | 가장 강력한 판매 메시지 전달: "당신은 돈을 잃고 있다." |

### **[STAGE 3] ✨ Value Uplift: PVU 계산 및 액션 유도 (The Gold Alert)**
*(시간 배분: 46초 ~ 90초)*

| 요소 | 내용 및 스토리보드 지시 | 시각적/기술적 구현 브리프 | 비즈니스 목표 |
| :--- | :--- | :--- | :--- |
| **솔루션 도입** | (내레이션) "Maru Engine은 이 단절을 찾아, 연결하고, 가치를 증폭합니다. 이것이 PVU입니다." 화면의 분위기가 급격히 밝고 깨끗한 톤으로 전환된다. (색상: 골드/네이비). | **애니메이션:** 'Missing Edge'가 발견된 지점을 향해 마치 레이저처럼 빛나는 연결선(Golden Edge)이 발사되어 노드들을 정확하게 이어준다. 이 과정 전체가 매끄러운 트랜지션으로 처리된다. | 해결책의 등장과 신뢰도 회복. |
| **PVU 계산** | (내레이션) "데이터 A와 B, 그리고 발견된 연결성 C를 곱하여 잠재 추가 순이익을 예측합니다." 화면 중앙에 `[PVU Formula: A x B x C]`가 명확한 그래프 형태로 나타나고, 각 변수가 실시간으로 채워지며 계산된다. | **UI/UX:** 간결하고 세련된 플로우차트 애니메이션. 수치가 빠르게 *증폭*하는 느낌(Exponential Growth)을 주어 기술적 우위를 강조한다. (이 과정은 복잡하므로 속도를 조절하여 집중력을 유지해야 함). | 핵심 지식 전달: 우리의 가치는 '연결'과 '계산'에 있다. |
| **최종 클라이맥스** | 계산 결과, 최종 PVU 수치가 이전의 손실액($Loss)보다 훨씬 커진 금액으로 확정된다. (예: Loss $3M $\rightarrow$ Potential Gain $15M). 이 과정에서 '골드 경고(Gold Alert)'가 터진다. | **애니메이션:** `✨ GOLD ALERT! ✨` 문구가 화면 전체를 덮으며, 가장 강력한 CTA 버튼이 하단에 고정된다. (Mockup 참조: 크레딧 잔여량 포함). **CTA Button Text:** "지금 연결하고 $15M의 기회를 잡으세요." | 즉각적이고 거부할 수 없는 행동 유도(Conversion). |
