## 📑 [경쟁 통합 분석 리포트] 주요 생산성 플랫폼 연동 전략 (Integration Points)

**작성 목적:** Maru Company의 STT 기반 자동화 기능을 Notion, Trello, Slack 등 주요 협업 툴에 어떻게 녹여낼지 기술적/구조적으로 정의하고, 성공적인 API 연결 고리(Integration Points)를 도출한다.
**분석 대상:** Notion (문서 구조), Trello (프로젝트 보드), Slack (실시간 워크플로우).

### 1. 플랫폼별 데이터 구조 및 관리 방식 분석

| 플랫폼 | 핵심 기능/데이터 구조 원칙 | 1인 창업자가 사용하는 패턴 | Maru Company 관점의 강점/약점 |
| :--- | :--- | :--- | :--- |
| **Notion** | **블록 기반 (Block-based)**: 모든 정보가 독립적인 '블록' 단위로 구성되며, 데이터베이스는 관계형(Relation) 구조를 가짐. 유연성이 극대화된 문서 관리 시스템. | 콘텐츠 기획서 작성 → DB에 아이디어 블록 저장 → 태그/관계 설정으로 체계화. (장문 자료의 구조화가 핵심) | **[강점]** STT 변환 텍스트 자체를 Notion 블록 단위로 자동 분할 및 포맷팅 가능. AI 요약 결과물을 '원천 정보'와 연결하여 DB 항목에 삽입 용이. |
| **Trello** | **보드/카드/리스트 구조 (Kanban)**: 작업 흐름(Flow) 중심의 시각적 관리. 카드 하나가 하나의 독립적인 업무 단위이자 진행 상태를 나타냄. | 아이디어 → 기획 중 → 검토 대기 → 완료와 같은 단계별 업무 추적. (단순한 Task 관리 및 프로세스 추적이 핵심) | **[강점]** 음성 녹취록을 업로드하고, 이를 '카드'에 첨부하여 내용을 구조화(STT→NLP). 카드 상태 변화 시 자동 리마인더/다음 액션 아이템 추출 기능 구현 가능. |
| **Slack** | **워크스페이스/채널 중심의 대화 흐름 (Conversation)**: 실시간 소통과 즉각적인 알림이 핵심. API를 통한 외부 서비스 연동 및 봇(Bot) 활용성이 높음. | 팀원 간 아이디어 브레인스토밍 → 회의록을 채널에 공유 → 액션 아이템을 담당자에게 태그하여 할당. (실시간 정보 분산과 연결이 핵심) | **[강점]** STT 변환 후, 가장 중요한 '핵심 요약'만 추출하여 지정된 채널에 자동 포스팅(Bot Action). 회의 중 녹음 → 즉시 스크립트 및 요약을 브리핑하는 워크플로우 구축 가능. |

### 2. Maru Company 서비스 통합 자산화 방안 (Integration Points)

Maru Company는 '단순한 변환기'가 아닌, **'생산성 생태계의 구조를 보강하는 지식 레이어(Knowledge Layer)'**로 포지셔닝해야 합니다.

| 플랫폼 | 통합 목표 및 가치 제안 | 구체적인 기술적 연결 고리 (Integration Points) | 핵심 키워드 |
| :--- | :--- | :--- | :--- |
| **Notion** | **원천 자산의 구조화:** 녹음된 음성 자료를 단순 텍스트로 변환하는 것이 아니라, '콘텐츠 아카이브 데이터베이스'에 바로 입력되는 자동화 워크플로우 제공. | 1. `[STT] API Trigger`: 외부 파일 업로드 시 STT 실행 → 2. `[NLP/AI]` 분석 결과(요약, 핵심 키워드)를 JSON 포맷으로 생성 → 3. Notion Block API로 해당 블록을 지정된 DB 관계에 자동 삽입. | **Atomic Content Block Injection**, Relation Mapping, Source Linkage (원천 링크 연결) |
| **Trello** | **프로세스 기반 자산화:** 추상적인 아이디어를 녹음하고 → 이를 '검토가 필요한 업무 카드'로 즉시 변환하여 워크플로우에 투입. | 1. `[Webhook]` Trigger: 음성 파일 업로드 시 자동 API 호출 → 2. `[AI/NLP]` 추출된 액션 아이템(To-do List)을 JSON 배열로 파싱 → 3. Trello Card Description 필드에 리스트 형태로 삽입 및 담당자 지정. | **Flow Automation**, Action Item Extraction, Kanban Integration, Due Date Mapping |
| **Slack** | **실시간 의사결정 지원:** 회의나 브레인스토밍 세션 종료 즉시, 모든 내용을 구조화하여 핵심 요약과 다음 스텝을 알림(Notification) 형태로 자동 배포. | 1. `[Bot Command]` Trigger: `/maru summarize` 명령 실행 → 2. STT 및 NLP 처리 후, '제목', '핵심 의사결정 (Decisions)', '다음 할 일 (Action Items)' 세 섹션으로 구조화하여 메시지 포맷팅. | **Ephemeral Summarization**, Bot Workflow Hook, Real-time Knowledge Capture, Decision Log Generation |

### 3. 결론 및 전략적 시사점 요약

1.  **핵심 가치:** Maru Company는 "정보의 변환"을 넘어 "**구조화된 지식의 흐름(Structured Flow of Knowledge)**"을 제공해야 합니다.
2.  **기술 초점:** 모든 통합은 API 기반으로 설계되어야 하며, 데이터 입력/출력 포맷을 **JSON 또는 YAML 스키마**로 정의하여 개발팀에 전달하는 것이 필수적입니다. (이는 '원자성(Atomicity)' 원칙과 일맥상통합니다.)
3.  **최우선 MVP 통합 목표:** 초기 버전에서는 Notion의 **'Atomic Content Block Injection'** 기능을 최우선으로 구현해야 합니다. 이는 가장 높은 가치 밀도(Value Density)를 가지며, 사용자가 '자료 관리'라는 본질적인 고통점과 직결되기 때문입니다.
