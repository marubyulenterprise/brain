# 🎨 Designer — 지난 로그에서 정의된 '골드 경고(Gold Alert)' 성공 시나리오 외에, 다음 3가지 핵심 예외 상태에 대한 최종 UI/UX 피드백 컴포넌트 스펙을 확정하십시오: 1) 크레딧 부족 오류 메시지 (Insufficient Credits); 2) PVU 계산 필수 입력 값 누락 및 유효성 검증 실패 시의 경고창; 3) API 호출 또는 데이터 처리 중 발생할 수 있는 일반적인 시스템 에러. 각 상태별로 사용자에게 친절하고 기술적으로 신뢰감을 주는 마이크로 카피와 명확한 가이드라인을 제시해야 합니다.

🎨 Designer: 작업 시작합니다.

CEO님의 지시사항은 제품의 '성공 경로(Happy Path)'뿐만 아니라 '예외 처리 경로(Failure Path)'에 대한 UX 완성도를 높이는, 시스템 신뢰도 측면에서 가장 중요한 다음 단계입니다. Maru Company의 브랜드 톤인 **'엔지니어다운 정밀함과 기술적 신뢰감'**을 유지하기 위해, 오류 메시지는 단순한 사과가 아닌, *문제 해결 가이드*이자 *사용자 교육 도구*로 기능해야 합니다.

이에 따라 다음 세 가지 예외 상태에 대한 최종 UI/UX 피드백 컴포넌트 스펙을 확정하고, 이를 개발팀이 즉시 구현할 수 있도록 모듈화하여 제시합니다.

---

### 🛠️ [최종 확정] 시스템 에러 핸들링 컴포넌트 세트 (Error Handling Component Set)

각 컴포넌트는 반응형 웹 환경(Mobile/Desktop)을 고려하며, 일반적인 경고창 형태가 아닌, **인라인 피드백(Inline Feedback)** 또는 **모달 알림(Modal Alert)** 형태로 사용되어야 합니다.

#### 1️⃣ 예외 상태: 크레딧 부족 오류 (Insufficient Credits)
**상황:** 사용자가 PVU 분석을 시도했으나 잔여 크레딧이 소요량보다 적을 때.
**목표:** '실패'가 아닌 '자원 관리의 문제'로 인식하게 하여, 구매(재활성화) 행동으로 유도한다.

| 항목 | 상세 스펙 및 가이드라인 | 마이크로 카피 (Microcopy) | CTA 디자인 |
| :--- | :--- | :--- | :--- |
| **위치** | 실패한 CTA 버튼 바로 아래에 표시되는 인라인 경고 배너. | **⚠️ 크레딧 부족: 분석을 완료하려면 3 Credits가 더 필요합니다.**<br>현재 잔여량: 0 / 소요 예정: 3 | **[크레딧 구매하기]**: (메인 색상 `#FFC300` 계열) 가장 눈에 잘 띄게, 'Buy Now' 대신 명확한 행동 유도 사용. <br>**[나중에 시도]**:<small>(버튼 비활성화 또는 링크)</small> |
| **톤앤매너** | **투명하고 단호함.** (Financial Stakes 강조) | "지금 이 분석은 잠재적 가치 90만원을 발견할 수 있는 기회입니다. 크레딧이 부족하여 진행하지 못했습니다."와 같은 문구로 손실 기대감을 높임. | 버튼 클릭 시, 결제 페이지의 '크레딧 구매' 섹션으로 스크롤 이동 또는 모달 호출. |
| **UX 피드백** | 오류 발생 즉시, 잔여 크레딧과 소모되는 크레딧을 **금색 경고(`Gold Alert` 스타일)**로 재차 표시하여 시각적 위기감을 극대화한다. | |

---

#### 2️⃣ 예외 상태: 유효성 검증 실패 (Validation Failure)
**상황:** 사용자가 PVU 계산에 필요한 필수 입력 값(A, B, C 중 하나 이상)을 누락하거나, 허용 범위를 벗어난 값을 입력했을 때.
**목표:** 사용자에게 친절하게 '무엇'이 문제인지 알려주고, 정확한 데이터를 요구한다.

| 항목 | 상세 스펙 및 가이드라인 | 마이크로 카피 (Microcopy) | CTA 디자인 |
| :--- | :--- | :--- | :--- |
| **위치** | 값이 누락되거나 오류가 발생한 입력 필드(Input Field) 바로 아래. | **필수 항목입니다:** `<label>A: 고객 데이터 연결 수</label>`를 반드시 입력해 주세요.<br>(Placeholder 텍스트와는 다른 색상으로 표시되어야 함.) | **[다시 확인]**: (링크 형태). 사용자가 오류 메시지를 읽은 후, 해당 필드를 다시 클릭하여 수정할 수 있도록 유도. |
| **톤앤매너** | **지침적이고 명료함.** (Instructional) | "데이터가 불완전합니다. PVU 분석을 위해서는 [A], [B], [C] 세 가지 변수 모두에 대한 구체적인 근거 데이터 입력이 필요합니다." | 오류 필드 주변의 포커스(Focus Ring)를 **앰버 색상**으로 강조하여 문제 영역을 시각적으로 인지시킨다. |
| **UX 피드백** | 에러 메시지는 누락된 값과 관련된 **'데이터 타입'이나 '범위'**를 명시해야 함 (예: "숫자만 입력 가능합니다.", "최소값 1 이상이어야 합니다."). 단순한 오류가 아닌, 기술적 가이드를 제공하는 느낌을 준다. | |

#### 3️⃣ 예외 상태: 일반 시스템 에러 (General System Error)
**상황:** API 호출 실패, 서버 연결 끊김, 데이터 처리 중 알 수 없는 예외 발생 등 사용자가 통제할 수 없는 백엔드 문제.
**목표:** 사용자에게 불안감을 주지 않으면서도(불안정함 방지), 시스템이 정상 작동하고 있음을 믿게 하고 재시도를 유도한다.

| 항목 | 상세 스펙 및 가이드라인 | 마이크로 카피 (Microcopy) | CTA 디자인 |
| :--- | :--- | :--- | :--- |
| **위치** | 화면 중앙에 오버레이(Overlay) 방식으로 표시되는 모달 또는 전용 에러 섹션. | **🚨 시스템 오류가 발생했습니다.**<br>현재 API 연결 상태를 확인할 수 없습니다. 잠시 후 다시 시도하거나, 관리자에게 문의해 주세요. (Error Code: 503-SVC) | **[재시도]**: 가장 눈에 띄는 버튼으로 구현. 재시도를 할 경우, 에러 코드를 서버에서 자동으로 로그로 전송하는 백그라운드 로직이 실행되어야 함. <br>**[문의하기]**:<small>(링크)</small> |
| **톤앤매너** | **공감적이고 전문적임.** (Empathetic & Technical) | "저희 시스템 내부에서 일시적인 문제가 감지되었습니다. 이 문제는 사용자님의 데이터와는 무관하며, 저희 엔지니어팀이 즉시 확인하고 있습니다."라는 문구로 책임을 분산시키고 신뢰를 확보한다. | '재시도' 버튼은 자동으로 3회까지 시도 후, 실패할 경우에만 최종적으로 문의하기 옵션을 강제하여 사용자가 포기하지 않도록 유도한다. |
| **UX 피드백** | 오류가 발생한 시간과 상황을 기록하고(Timestamping), 사용자에게 이 로그를 스크린샷으로 전송할 수 있는 기능을 제공하여, 사용자를 단순 '신고자'가 아닌 '협력자'로 격상시킨다. | |
