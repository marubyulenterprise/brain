CEO님의 지시사항을 기반으로, 완성된 프로토타입의 기능적 가치(데이터 결핍 충격 자각)를 가장 잘 수익화할 수 있는 단계별 비즈니스 모델을 재정립했습니다. 핵심 전략은 **'무료 진단 $\to$ 손실 인식($L$) $\to$ 유료 해결책 제시'**의 흐름에 집중하는 것입니다.

## 💰 MVP 수익 모델 제안: Missing Edge Shock Component (V1.0)

### 🎯 핵심 비즈니스 로직 (The Core Logic)
우리는 단순한 '도구'를 파는 것이 아닙니다. **"지금 당장 당신이 인식하지 못하는 재무적 손실액($L$)을 포착하고, 그것을 회복할 수 있는 잠재 가치(PVU)"**를 판매합니다. 모든 가격 모델은 이 $L$ Amount의 공포심에서 출발해야 합니다.

### 📈 3가지 단계별 가격 책정 세트 (Pricing Sets)

#### 💎 Set 1: 크레딧 기반 모델 (The Utility Model - 개발자/기술 사용자 타깃)
*   **타겟 페르소나:** IT 개발자, 데이터 분석가 등 기능의 *구체적인 사용량*을 따지는 기술적 사용자.
*   **Pain Point:** "내가 원하는 기능을 쓰려면 몇 번이나 시도해야 하는지" (사용성/제한).
*   **구조:** Pay-as-you-go 방식과 기본 구독료를 결합하여, **'노동 투입량 대비 가치'**에 초점을 맞춥니다.

| 등급 | 가격 책정 | 주요 기능 제한 및 경험 설계 | 상향 전환 유도 장치 (Upsell Hook) |
| :--- | :--- | :--- | :--- |
| **Free Tier** | 무료 (0원) | ⚡️ 'Missing Edge' 진단 1회 제한. / 결과 보고서: $L$ Amount만 제시 (수치만). / 구조화된 데이터 추출 불가. | "이 손실액($L$)을 해결하려면, 최소 [N]개의 크레딧이 필요합니다."라는 명시적 안내와 함께 CTA 배치. |
| **Basic Tier** | 월 구독료 ($X$원) | 📚 진단 횟수 무제한. / 추출 가능한 데이터 양: 제한 (예: 월 50,000 토큰). / 크레딧 제공량: 소량 지급 (예: 매월 20 크레딧). | **[프리미엄 전환]** '전체 원본 출처 통합(Source Integration)' 기능은 Premium 전용으로 막아둠. |
| **Premium Tier** | 월 구독료 ($Y$원) | 👑 모든 사용 가능. / 무제한 크레딧 및 API 접근 권한 제공. / 데이터 소스 확장 (외부 DB 연동). / *가장 중요:* $L$ Amount의 원본 데이터를 역추적하여 보고서에 포함. | "더 큰 규모로 자동화를 구축하고 싶다면, 기업용 문의를 남겨주세요." → Enterprise Tier 유도. |

#### 🚀 Set 2: 가치 기반 모델 (The Efficiency Model - 일반 1인 창업자/마케터 타깃)
*   **타겟 페르소나:** 시간 절약이 가장 중요한 마케터, 컨설턴트 등 비즈니스 프로세스 효율화가 필요한 사용자.
*   **Pain Point:** "반복되는 수작업에 너무 많은 시간이 낭비된다" (시간/효율).
*   **구조:** **'처리할 콘텐츠의 양(Volume)'**과 **'얻게 될 시간 가치(Time Value)'**를 기준으로 과금합니다.

| 등급 | 가격 책정 | 주요 기능 제한 및 경험 설계 | 상향 전환 유도 장치 (Upsell Hook) |
| :--- | :--- | :--- | :--- |
| **Free Tier** | 무료 (0원) | 🔍 '진단 보고서'만 제공. / 문제 인식: "이 데이터를 수동으로 구조화하는 데 최소 [N]시간이 소요됩니다."라는 시간 손실액($L_{Time}$) 제시. | CTA: "자동화를 원하시면, Premium에서 **[10시간 절약 패키지]**를 구매하세요." (가상의 시간을 돈으로 환산). |
| **Pro Tier** | 월 구독료 ($A$원) | ⏱️ 시간당 가치를 기준으로 크레딧 제공. / 처리 가능한 콘텐츠 볼륨: 중간 수준 제한 (예: 월 10개 프로젝트/300만 단어). / 주요 기능 사용 가능. | **[프리미엄 전환]** '프로세스 최적화 시나리오 빌더'와 같은 고급 자동화 워크플로우는 Premium 전용으로 막아둠. |
| **Agency Tier** | 월 구독료 ($B$원) | 🏢 무제한 볼륨 및 팀 계정 지원 (최대 사용자 수 제한). / 통합 관리 대시보드 제공. / *핵심:* $L_{Time}$을 최소화하는 맞춤형 '프로세스 설계 컨설팅' 포함. | "우리 회사의 특수 산업군 데이터에 최적화된 커스터마이징은 별도 협의가 필요합니다." → 기업 매출 규모에 연동한 Custom Pricing 유도. |

#### 🏗️ Set 3: 자산 기반 모델 (The Ownership Model - 에이전시/대형 팀 타깃)
*   **타겟 페르소나:** 다수의 데이터를 보유하고 있으며, 이를 **'지적 자산(IP)'**으로 관리하고자 하는 조직.
*   **Pain Point:** "우리 회사 데이터의 가치 전체를 시스템화하여 소유하고 싶다" (통제력/소유권).
*   **구조:** 초기 투자금 성격이 강하며, **'데이터 통제권을 확보하는 것'**을 핵심 가치로 둡니다.

| 등급 | 가격 책정 | 주요 기능 제한 및 경험 설계 | 상향 전환 유도 장치 (Upsell Hook) |
| :--- | :--- | :--- | :--- |
| **Free Tier** | 무료 (0원) | 진단 보고서만 제공하며, 데이터 소유권/접근 권한은 전적으로 Maru Company에 있음. | "완전한 자산화 및 통제권을 원하시면, 자체 온프레미스 배포(On-Premise Deployment)를 고려해야 합니다." → 고가치의 맞춤형 요구사항 제시. |
| **Enterprise Tier** | 연간 계약 (고액 $C$원) | 🛡️ 데이터 소유권 및 API 완전 개방. / 자체 서버 구축 옵션 (온프레미스). / 전담 기술 지원팀 배정. / *핵심:* 모든 구조화된 데이터를 통합 지식 자산으로 관리하는 'Master Index' 기능 제공. | 계약 규모에 따라 추가 모듈(예: 특정 산업군 특화 NLP 모델)을 패키지로 판매하여 매출 증대. |
