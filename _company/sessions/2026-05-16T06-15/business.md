# 💰 Missing Edge 기반 수익화 전략: Lost Opportunity Cost 모델

## 1. 가격 책정 원칙 재정립 (Pricing Philosophy Shift)
기존의 크레딧 소모 방식(Input-Based Pricing)은 사용자가 얼마나 많이 '작업'했는지에 초점을 맞춥니다. 하지만 우리 제품의 핵심 가치는 **사용자 자신이 인식하지 못했던 중요한 연결고리나 잠재적 위험을 예측**하는 데 있습니다 (Output-Based Value).

따라서 가격 책정의 중심축을 다음으로 이동해야 합니다:
> **[기존]** $\text{Usage Cost} = \sum (\text{Feature } A \times \text{Credits Used})$
> **$\Downarrow$**
> **[변경]** $\text{Subscription Value} = \text{Number of Predicted Missing Edges} \times \text{Predicted Lost Opportunity Cost (PLC)}$

이 원칙에 따라, 각 티어는 '발견된 잠재적 위험의 규모'를 지표로 삼고, 이 위험을 제거하는 데 필요한 서비스임을 강조해야 합니다.

## 2. 가격 구조 시나리오 제안 (3-Tiered Pricing Model)

| 구분 | Tier 이름 | 타겟 고객 | 핵심 가치 Proposition | 주요 제한점 (Value Wall) |
| :--- | :--- | :--- | :--- | :--- |
| **Free** | **Starter Edge** | 호기심 있는 초보자, 개인 학습자 | "내 지식의 뼈대를 확인하세요." (Discovery) | *Edge 경고*만 가능. 발견된 Missing Edge에 대한 **구체적인 해결책(Action Plan)** 및 **정량화된 PLC 계산 불가**. 월 최대 분석 데이터 볼륨 제한 (예: 5개 소스). |
| **Pro** | **Growth Shield** | 1인 창업가, 콘텐츠 제작자, 마케터 | "잠재적 손실을 구조적으로 방어하세요." (Prevention) | Missing Edge를 발견하고, 관련된 **핵심 컨셉/키워드(Nodes)**까지 제안 가능. PLC 계산은 '예상치' 수준으로만 제한됨 (Max 10개 분석). |
| **Enterprise** | **Autonomous Predictor** | 중소기업, 팀 단위 프로젝트 팀 | "시스템적 위험을 예측하고 최적의 로드맵을 설계하세요." (Prediction & Optimization) | 무제한 데이터 소스 연결 및 통합 관리. Missing Edge 발견 시 **최종적인 PLC 수치 계산(Historical Data 기반)** 및 **자동화된 해결 액션 플랜** 제공. 전담 API 접근 권한. |

## 3. 티어별 상세 내용 및 KPI 정의

### A. Starter Edge (Freemium - 가치 벽 설정)
*   **가격:** 무료 ($\text{₩}0$)
*   **주요 기능:** 지식 그래프 시각화 기본 보기, 기본적인 관계성(Edge) 연결 탐색, 간단한 Missing Edge 알림(앰버 경고).
*   **가치 벽 (Value Wall):** 사용자가 "어딘가 부족하다"는 감만 느끼게 합니다. 가장 중요한 것은 **'왜' 부족하며 '얼마나' 큰 손실인지**에 대한 정량적 답을 주지 않습니다. (예: "이 관계성은 누락되었습니다." $\rightarrow$ *[필수 유료 기능] 이 연결고리가 놓칠 경우, 예상 매출 $X$만큼의 기회가 상실될 수 있습니다.*)
*   **KPI 목표치:**
    *   **Sign-up to Feature A Usage Rate (가입자 중 그래프 보기 비율):** 80% 이상.
    *   **Activation Metric (핵심 행동):** Missing Edge 경고 알림을 최소 1회 확인한 사용자 비율 (25%).
    *   **Conversion Goal:** 사용자가 "Missing Edge 발견 $\rightarrow$ PLC 궁금증 발생"의 경험을 하도록 유도하여 Pro Tier로 전환.

### B. Growth Shield (Pro - 효율성 및 방어)
*   **가격:** 월 구독료 ($\text{₩}49,000 \sim 69,000$)
*   **주요 기능:** Missing Edge 발견 + **해결을 위한 핵심 노드/키워드 추천**, 제한적 PLC 분석 (예: 최근 3개월 데이터 기반 예측). 원천 콘텐츠 재가공 크레딧 추가 제공.
*   **가치 벽 (Value Wall):** '개인적인 문제 해결' 범위에 머무릅니다. 시스템 전체의 복잡성이나 대용량/장기적 추세를 다루지 못합니다. PLC는 **"최대 잠재 손실액 $\pm 20\%$ 오차 범위 내 예상치"**로 제한됩니다.
*   **KPI 목표치:**
    *   **Usage Depth (사용 깊이):** 그래프 분석 후, 최소 3개 이상의 '추천 노드'를 추가한 사용자 비율 (45%).
    *   **Retention Rate:** 이탈 방지 기능(예: 자동 리마인더) 사용률 증가.
    *   **Conversion Goal:** 사용자가 "수동적인 예측치로는 부족하다"는 인식에 도달하여 Enterprise Tier의 '정확성'을 요구하도록 유도.

### C. Autonomous Predictor (Enterprise - 완전한 통제 및 최적화)
*   **가격:** 맞춤형/사용량 기반 ($\text{₩}199,000+$)
*   **주요 기능:** 무제한 데이터 소스 통합 연결, 실시간 Lost Opportunity Cost(PLC) 정밀 계산 (과거 성공/실패 사례 데이터 기반), Missing Edge 발생 시 **자동화된 액션 플랜 및 개발 백로그 제안**. 전담 API 연동.
*   **가치 벽 (Value Wall):** 이 단계에서는 '벽'이 아니라 '진입 장벽'입니다. 복잡한 온보딩, 커스터마이징, 그리고 초기 구축 비용 자체가 높은 진입 장벽 역할을 합니다.
*   **KPI 목표치:**
    *   **High-Value Feature Adoption (핵심 기능 채택):** 자동화된 액션 플랜을 실제로 업무에 반영하는 고객사 비율 (70% 이상).
    *   **LTV (Lifetime Value):** 초기 도입 비용 대비 예측되는 시간/비용 절감액의 월 평균 $X$배를 상회하도록 설계.
