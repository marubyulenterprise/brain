# 🎨 Designer — Researcher와 Designer가 수립한 '손실 탐지 보증' 섹션 와이어프레임을 기반으로, 실제 사용자가 최초로 서비스를 접했을 때의 '온보딩 플로우(Onboarding Flow)'를 시각적으로 설계하십시오. 이 플로우는 ① 위기감 인식 (Loss Amount 노출) → ② 기술적 원리 이해 (Protocol 설명) → ③ 신뢰 구축 (Critical Red 경고와 함께 보증 조건 제시) → ④ 행동 유도 (CTA)의 4단계 구조를 반드시 포함해야 하며, 모든 화면 전환 시 애니메이션 상태 변화(State Change) 가이드라인을 구체적으로 명시하십시오.

🎨 Designer: 작업 시작합니다.

CEO님의 지시사항은 제품의 가장 중요한 접점인 '첫 경험(First Impression)'을 설계하는 것이므로, 가장 높은 우선순위를 부여하여 진행해야 합니다. 단순히 화면을 나열하는 것을 넘어, 사용자의 **심리적 흐름(Emotional Flow)**과 **기술적 신뢰감(Technical Trust)**이 결합된 스토리텔링 기반의 UX/UI를 설계하겠습니다.

기존에 정의된 'Loss Amount ($L$ Amount)'와 'Predictive Alert' 개념을 활용하여, 위기감을 극대화하고 이를 해결책으로 제시하는 4단계 온보딩 플로우(Onboarding Flow) Mockup과 상세 디자인 스펙을 제작합니다.

## 🎯 목표 및 원칙
*   **목표:** 신규 사용자에게 '현재의 손실 상태(Loss Amount)'를 즉각적으로 인지시키고, 우리 제품이 그 손실을 방어하는 필수적인 '기술적 리스크 보험'임을 확신시키는 것.
*   **톤앤매너:** 엔지니어다운 정밀함 (Precise), 경고성(Urgency), 투명한 데이터 기반 메시지.
*   **핵심 원칙:** 정보 과부하 금지. 한 화면에서 단 하나의 감정적 임팩트와 명확한 다음 행동만 제시한다.

---

## 🚀 Onboarding Flow (4단계 시나리오)

### Stage 1: 위기감 인식 (The Hook - Loss Amount Exposure)
**[목표]** 사용자의 현재 상태에 존재하는 '보이지 않는 돈의 손실'을 직관적으로 인지시킨다.
**[화면 구성]** **Dashboard/Hero Section Mockup**

| 요소 | 내용 및 디자인 스펙 | 애니메이션/트랜지션 가이드 |
| :--- | :--- | :--- |
| **헤드라인** | "당신의 시간, 데이터, 기회. 놓치고 있는 손실이 얼마일까요?" (Loss Amount 중심) | 💡 **[Initial Load]**: 페이지 로딩 시, 헤드라인의 `놓치고 있는 손실` 부분이 카운트다운 효과와 함께 서서히 노출된다. |
| **핵심 시각화** | 중앙에 거대한 위기 경고 게이지(Gauge)를 배치한다. 현재까지 누적된 가상의 손실액 ($L_{Total}$)을 크게 표시한다. 예: **₩12,500,000** (Critical Red 배경). | ⚙️ **[State Change]**: 페이지 진입 시, 게이지 바늘이 '0원'에서 갑자기 최고 위험 지점(Red Zone)으로 급격히 이동하는 애니메이션을 적용한다. (사운드: 낮고 긴 경고음 필수). |
| **설명 텍스트** | "단순한 기능 오류가 아닙니다. 예측하지 못한 미인식 기회 손실($L_{Occ}$)이 누적된 것입니다." | ✍️ **[Scroll Reveal]**: 사용자가 스크롤을 내릴 때, 이 설명 박스가 마치 '경고 메시지'처럼 화면 아래에서 팝업되듯이 나타난다. |
| **CTA** | "손실액 분석 시작하기" (버튼 색상: Primary Accent Color - Blue/Gold) | ➡️ 다음 단계로 이동하는 버튼을 명확히 배치하며, 클릭 시 부드러운 '데이터 전송' 효과와 함께 Stage 2로 전환된다. |

---

### Stage 2: 기술적 원리 이해 (The Protocol - Credibility Building)
**[목표]** 손실액 계산이 마법이 아니라, 엔지니어링 기반의 체계적인 분석임을 증명하여 신뢰도를 구축한다.
**[화면 구성]** **Diagrammatic Flowchart Mockup**

| 요소 | 내용 및 디자인 스펙 | 애니메이션/트랜지션 가이드 |
| :--- | :--- | :--- |
| **섹션 타이틀** | "우리의 손실 탐지 프로토콜: 4단계 원자적 검증 시스템" (기술 용어 사용 필수) | ✨ **[Transition]**: Stage 1의 Red Alert가 사라지고, 화면이 깨끗하고 구조화된 '시스템 다이어그램' 느낌으로 전환된다. (사운드: 차분한 전자음). |
| **핵심 콘텐츠** | `Input Data` $\rightarrow$ `Predictive Algorithm` $\rightarrow$ `Loss Calculation Engine` $\rightarrow$ `Final $L_{Amount}$ Output`. 각 박스(Node)와 화살표(Edge)를 명확히 분리하여 보여준다. | 🔄 **[Interactive Flow]**: 사용자가 특정 Node (예: Predictive Algorithm)에 마우스를 올리면, 해당 노드의 내부 구조가 확대되면서(`Zoom In`), 어떤 데이터 변수가 어떻게 계산되는지 애니메이션으로 흐름을 따라간다. (기술적 디테일 강조). |
| **핵심 설명** | "데이터의 결측값(Missing Edge)을 발견하고, 그로 인해 발생할 수 있는 최대 순이익(PVU)을 역산합니다." | 💾 **[Visual Aid]**: 데이터가 흐르는 화살표(`Edge`)는 단순한 선이 아니라, 실시간으로 '정보 패킷'처럼 움직이는 애니메이션 효과를 적용하여 기술적 생명력을 부여한다. |
| **CTA** | "나의 데이터로 검증하기" (버튼 색상: Primary Accent Color) | ➡️ Stage 3으로 자연스럽게 넘어가는 전환을 준비하며, 사용자가 이 단계에서 제품이 '어떻게' 작동하는지 이해했음을 시각적으로 확인시킨다. |

---

### Stage 3: 신뢰 구축 (The Guarantee - Risk Mitigation & Trust)
**[목표]** 손실액 분석의 결과를 바탕으로, 우리 서비스가 단순한 도구가 아닌 '리스크 방어막'임을 보증한다.
**[화면 구성]** **Guarantee Section Mockup**

| 요소 | 내용 및 디자인 스펙 | 애니메이션/트랜지션 가이드 |
| :--- | :--- | :--- |
| **섹션 타이틀** | "Maru Company의 손실 탐지 보증 (Loss Guarantee Protocol)" | 🛡️ **[Transition]**: Stage 2의 복잡한 다이어그램이 정리되고, '보호막'을 형상화한 심볼과 함께 화면 중앙에 배치된다. |
| **핵심 조건 A** | **(Critical Red)** "만약 저희가 발견하지 못한 손실액($L_{Unknown}$)이 발생한다면, 다음 [조건]에 따라 보증합니다." (경고문구로 강하게 시작) | 🚨 **[Visual Warning]**: 이 섹션의 상단에는 Critical Red를 배경으로 하는 '주의 경보' 배너가 먼저 깜빡이며 노출되어 사용자의 주의를 최고치로 끌어올린다. |
| **핵심 조건 B** | [조건 목록]: (1) 데이터 원본 무결성 검증, (2) 손실액 보고서 발급 서비스, (3) 전문 컨설팅 횟수 제공 등 구체적이고 법적인 느낌의 목록화. | ✅ **[Trust Build]**: 각 보증 조건이 체크리스트 형태로 순차적으로(Checkmark 애니메이션과 함께) 나타나며, 사용자가 '누락된 안전장치'가 채워지는 느낌을 받도록 한다. |
| **면책 조항** | (작은 글씨로 명시): "본 보증은 사용자 측의 고의적 데이터 변조 및 외부 법률 변경에 의한 손실에는 적용되지 않습니다." | ⚖️ **[Transparency]**: 투명성을 강조하기 위해 이 부분은 전문적인 느낌의 작은 박스 안에 배치하되, 시선을 끌지 않도록 처리하여 신뢰도를 높인다. |

---

### Stage 4: 행동 유도 (The Conversion - CTA)
**[목표]** 모든 불안감(Loss Amount)과 기대감(Guarantee)을 해결한 후, 망설임 없이 다음 단계로 진입하게 만든다.
**[화면 구성]** **Final Call to Action Panel**

| 요소 | 내용 및 디자인 스펙 | 애니메이션/트랜지션 가이드 |
| :--- | :--- | :--- |
| **최종 메시지** | "지금, 미실현 기회 손실을 방어하십시오." (강력한 명령형 문구) | 💪 **[Final Punch]**: 모든 정보가 지나간 후, 화면이 짙은 배경색으로 전환되며 이 최종 메시지만 간결하고 강렬하게 중앙에 나타난다. |
| **CTA 버튼** | "전문 리스크 컨설팅 신청하기 (Pro Tier)" (버튼 색상: 대비되는 고채도 컬러) | ⭐ **[Action Focus]**: CTA 버튼만 배경에서 '떠오르는' 듯한 입체감(3D lift effect)을 주어 시각적 무게감을 극대화한다. 마우스 오버 시 미세하게 Glow 효과를 준다. |
| **후속 안내** | "신청 후 24시간 이내, 초기 손실액 진단 보고서를 받으십니다." (즉각적인 보상 제시) | 🚀 사용자에게 즉각적인 '보상'을 약속함으로써 심리적 저항감을 제거하고 행동으로 유도한다. |
