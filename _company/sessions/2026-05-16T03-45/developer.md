## 💾 개발 백로그: Knowledge Graph Builder MVP 통합 테스트 케이스 (v1.0)

**목표:** 원천 데이터 업로드부터 구조화된 지식 자산 추출까지의 전 과정(Critical Path)에 대한 안정성 검증 및 API 명세 확정.
**범위:** 3개의 마이크로서비스 엔드포인트 통합 테스트.

### 1️⃣ Endpoint: `/api/input-data` (데이터 유입 및 준비)

| 항목 | 내용 |
| :--- | :--- |
| **목적** | 비구조화된 대량의 원본 콘텐츠(Raw Text Corpus)를 수신하고, 내부적으로 청크 단위로 분할하여 임시 저장소에 등록한다. (데이터 전처리/세그먼테이션). |
| **메서드** | `POST /api/input-data` |
| **요청 조건 (Request Body)** | `{"files": [ { "file_id": "uuid", "mime_type": "text/plain", "content_base64": "..." }, ... ], "user_context": {"user_id": 123, "source_name": "회의록" }}` |
| **성공 시나리오 (Happy Path)** | - **응답 코드:** `202 Accepted` <br> - **예상 응답:** `{"status": "Processing", "job_id": "uuid-123", "message": "파일 전처리가 시작되었습니다. 주기적으로 상태를 확인해주세요."}` <br> - **기술 검증:** 파일별 청크 분할 및 메타데이터(Source/Timestamp)가 정확하게 저장되었는지 DB에서 확인 가능해야 함. |
| **실패 시나리오 (Failure Cases)** | 1. **[파일 형식 오류]**: MIME 타입이 지원하지 않는 경우 (`200 Bad Request`).<br> 2. **[전송 용량 초과]**: 총 데이터 크기가 시스템 제한을 초과하는 경우 (`413 Payload Too Large`).<br> 3. **[인증 실패]**: `user_id` 또는 토큰이 유효하지 않은 경우 (`401 Unauthorized`). |
| **트랜잭션 원칙** | 비동기 작업(Async Job)으로 처리하며, 즉각적인 트랜잭션 실패는 발생하지 않음. 성공 시 Job ID를 반환하여 상태 추적을 보장해야 함. |

### 2️⃣ Endpoint: `/api/discover-edge` (핵심 가치 발견 및 크레딧 소모)

| 항목 | 내용 |
| :--- | :--- |
| **목적** | 전처리된 청크 데이터를 분석하여, 문장/개념 간의 논리적 연결고리(Edge)를 추론하고 지식 그래프 구조를 생성한다. (가장 중요한 유료 로직). |
| **메서드** | `POST /api/discover-edge` |
| **요청 조건 (Request Body)** | `{"job_id": "uuid-123", "input_data_source": "uuid-123"}` |
| **성공 시나리오 (Happy Path)** | - **응답 코드:** `200 OK` <br> - **예상 응답:** `{"status": "Success", "credits_consumed": 5.0, "graph_summary": {"nodes": 15, "edges": 32}, "output_data_ref": "uuid-456"}` <br> - **기술 검증:** DB에 Edge 리스트(Source Node ID $\to$ Relationship Type $\to$ Target Node ID)가 원자적 트랜잭션으로 기록되었는지 확인. |
| **실패 시나리오 (Failure Cases)** | 1. **[크레딧 부족]**: 시스템이 추정하는 비용보다 사용자의 잔여 크레딧이 적은 경우 (`402 Payment Required`). *가장 중요.*<br> 2. **[입력 데이터 부재]**: `job_id`로 참조한 원본 데이터가 이미 삭제되었거나 유효하지 않은 경우 (`404 Not Found`).<br> 3. **[추론 실패]**: NLP 모델 실행 중 예외 발생 (예: 메모리 부족, 네트워크 문제) 시 (`500 Internal Error`). |
| **트랜잭션 원칙** | **최우선 순위:** 크레딧 검증 $\to$ 트랜잭션 시작 $\to$ 그래프 생성 및 DB 기록 $\to$ 성공적 결과 반환. 만약 트랜잭션 중단 시, *반드시* 모든 변경 사항을 롤백하고, 실패 사유와 함께 `500` 응답 코드를 돌려줘야 함. (ACID 원칙 준수) |

### 3️⃣ Endpoint: `/api/finalize-graph` (결과물 가공 및 출력)

| 항목 | 내용 |
| :--- | :--- |
| **목적** | `discover-edge`에서 생성된 구조화된 그래프 데이터를 최종 사용자가 보기 쉬운 형태(예: JSON, Mermaid.js 포맷)로 변환하고, 필요한 경우 요약 보고서를 함께 생성한다. |
| **메서드** | `POST /api/finalize-graph` |
| **요청 조건 (Request Body)** | `{"output_data_ref": "uuid-456"}` |
| **성공 시나리오 (Happy Path)** | - **응답 코드:** `200 OK` <br> - **예상 응답:** `{"status": "Complete", "exportable_format": {"mermaid": "graph TD...", "json": [...]}, "report_summary": "핵심 발견 엣지: 32개. 잠재적 주제 확장 방향 제시."}` <br> - **기술 검증:** 반환된 데이터 구조가 사전에 정의한 JSON 스키마를 만족하는지 유효성 검사(Validation)가 필수임. |
| **실패 시나리오 (Failure Cases)** | 1. **[데이터 미완성]**: `output_data_ref`에 연결된 그래프 데이터가 불완전하거나 누락된 경우 (`400 Bad Request`).<br> 2. **[권한 문제]**: 해당 결과를 열람할 권한이 없는 사용자에게 요청된 경우 (`403 Forbidden`). |
| **트랜잭션 원칙** | 읽기 전용(Read-Only) 작업에 가까우므로, 트랜잭션 실패보다는 데이터 정합성 검증과 포맷팅 로직의 견고성에 집중해야 함. |
